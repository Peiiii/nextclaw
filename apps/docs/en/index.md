## What happens inside a task

Provide an outcome, source material, and a working directory. The agent can use models, skills, a browser, the terminal, and local files. You can observe, redirect, or stop it, then inspect the actual artifact in the same workspace.

<div class="nc-docs-lifecycle">
  <div><span>1</span><strong>Describe the deliverable</strong><p>State the material, output format, destination, and boundaries.</p></div>
  <div><span>2</span><strong>Let the agent act</strong><p>It reads files, invokes tools, runs commands, opens the web, or delegates subtasks.</p></div>
  <div><span>3</span><strong>Stay in control</strong><p>Watch actions, answer questions, add constraints, or stop execution.</p></div>
  <div><span>4</span><strong>Inspect the result</strong><p>Open replies, files, source, Markdown, HTML, images, or a Panel App.</p></div>
</div>

## Run work on schedule or from a messaging app

Scheduled tasks cover briefs, checks, and recurring summaries. Channels let requests and results move through Weixin, Feishu, Telegram, Slack, and other familiar entry points. Prove the work locally before automating it.
