# NextClaw 路线图

本文档描述 NextClaw 当前阶段的具体优先级与执行方向。所有工作服务于 [产品愿景](/zh/project/vision)。

如果你想看一个更直观的公开页面，了解最近发版节奏、代码规模变化、产品笔记和当前界面截图，先看 [Project Pulse](/zh/project/project-pulse)。

## 当前阶段优先级

### 1. 自感知能力

- Agent 能查询自身版本、配置、运行状态、健康度、能力边界与文档入口
- 涵盖静态信息与动态信息
- 能在对话中回答“当前模型是什么”“我健康吗”“我有哪些渠道”

### 2. 自治能力

- Agent 可通过工具执行关键管理动作：改配置、启停渠道、增删 cron、安装/卸载插件、诊断/重启
- 高风险操作保留确认机制
- 用户通过自然语言完成原本需要 UI/CLI 的管理动作

### 3. 插件 SDK 与开发流程

- 稳定插件 SDK 与 OpenClaw 兼容的开发流程
- 明确内核与插件边界
- 支持从 UI 中安装、启用、配置、卸载插件

### 4. 开箱即用体验

- 首次启动展示核心能力与典型场景
- 预置场景模板，支持一键启用
- 优化从安装到首个可用场景的路径

## 下一阶段方向

### Marketplace 生态

- 用户插件/Skill 的发布与发现流程
- 评价、版本管理、依赖解析等基础设施
- 形成“使用 -> 开发 -> 分享 -> 发现”闭环

### 宿主系统与数据能力

- 文件、进程、系统监控等宿主工具能力
- 搜索、抓取、API 调用、信息聚合等互联网接入
- 整合散落在不同 App/平台中的用户数据

### 多 Agent 与多实例

- 多 Agent：会话隔离、bindings、路由与运行时语义增强
- 多实例：部署模型、配置边界与最佳实践沉淀

## 持续关注

- Landing 页面与主入口表达
- UI 与交互体验
- OpenClaw 兼容性
- Cron / Automation
- 可观测性与运维
- 文档与教程完整路径
