# 更新笔记

如果你想知道“最近有什么值得我关注或升级的变化”，从这里开始最合适。

`更新笔记` 面向用户，而不是面向底层实现细节。它强调：

- 这次更新对你有什么价值
- 你应该怎么用
- 想继续深入时去看哪里

## 入口

- [产品更新笔记](/zh/notes/)

## 它和别的页面有什么区别

- `更新笔记`：短更新，先讲用户价值
- [Project Pulse](/zh/project/project-pulse)：看整体交付节奏和公开项目脉搏
- [路线图](/zh/project/roadmap)：看接下来重点做什么
- [博客](/zh/blog/)：看更长的判断、取舍和设计思路
