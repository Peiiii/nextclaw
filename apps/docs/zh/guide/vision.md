# NextClaw 产品愿景

NextClaw 的目标是构建一个 **自知自治、可无限拓展、为数字世界全能而生** 的 AI Agent 平台。

以下四大核心支柱定义了产品的长期方向与根本信念。它们不是某个版本的功能清单，而是贯穿每一次迭代的北极星。

---

## 支柱一：自知与自治 —— 对自身系统"全知全能"的 AI

Agent 对自身所处的系统拥有完整的认知与控制力，真正做到"全知全能"。

**自知（全知）**

- Agent 知晓关于自身的一切：身份（NextClaw）、版本、完整配置（providers、models、channels、cron、plugins）、运行状态、健康度、能力边界与相关文档入口。
- 能在对话中回答任何关于自身的问题——"我有哪些渠道？"、"当前用的什么模型？"、"我健康吗？"、"我支持哪些能力？"
- 涵盖静态信息（配置、版本）与动态信息（运行时状态、实时健康度、会话上下文）。

**自治（全能）**

- Agent 可通过工具执行系统中的一切管理动作：修改配置、启停渠道、增删 cron 任务、安装/卸载插件、触发诊断与重启等。
- 高风险或不可逆操作需明确确认机制，确保安全可控。
- 内置聊天界面成为"自治系统"的一等入口——用户通过自然语言即可完成原本需要 UI/CLI 的所有操作。

**目标**：让 Agent 成为自身系统的完全代理，用户只需对话，即可了解和掌控 NextClaw 的一切。

---

## 支柱二：插件化拓展与生态繁荣

系统内核趋于稳定后，通过插件与 Skill 生态实现功能的无限拓展和社区自然增长。

**稳定的内核**

- 核心运行时、插件 SDK、渠道协议逐步收敛并保持向后兼容。
- 明确内核与插件的边界：内核负责调度、安全、生命周期管理；功能拓展交给插件。

**开放的插件/Skill 体系**

- 兼容 OpenClaw 标准的插件与 Skill 开发流程。
- 支持从 UI 中一键安装、启用、配置、卸载插件。
- 开发者可以用 NextClaw 本身开发插件，降低创作门槛。

**Marketplace 生态**

- 用户开发的优质插件和 Skill 可发布到 Marketplace 生态池。
- 实现"使用 → 开发 → 分享 → 发现"的闭环，驱动社区自然增长与生态繁荣。
- 完善评价、版本管理、依赖解析等基础设施。

## 支柱三：数字世界全能基础设施

NextClaw 不只是一个聊天机器人——它致力于成为用户在数字世界中"全能"的基础设施，体现在三个维度：

**a. 宿主系统的全能管理与充分利用**

- Agent 充分理解和利用所运行的宿主系统（操作系统、文件系统、本地服务、硬件资源等）。
- 通过工具调用实现对宿主系统的管理操作：文件管理、进程管理、系统监控、本地应用联动等。
- 让 AI 成为用户操控本地环境的智能中枢。

**b. 互联网数据与云计算能力的便捷利用**

- 便捷接入互联网数据源：搜索、抓取、API 调用、信息聚合。
- 调度和利用云计算资源：云存储、云函数、远程服务等。
- 让用户通过自然语言即可触达互联网世界的数据与算力，无需手动操作复杂工具。

**c. 用户数据的充分管理**

- 管理用户的本地数据：文件、笔记、配置、个人知识库。
- 连接和整合用户散布在各 App 和平台中的数据（邮件、日历、社交、笔记、云盘等）。
- 在尊重隐私与安全的前提下，为用户提供统一的数据视图与操作能力。

## 支柱四：开箱即用

降低一切上手门槛，让用户安装后立刻知道能做什么、并能一键开始使用。

**即刻可见的能力**

- 首次启动即展示 NextClaw 的核心能力和典型场景（定时提醒、智能问答、多渠道接入、数据管理等）。
- 提供"能力画廊"或引导式向导，让用户直观感受产品价值。

**一键启用**

- 关键场景支持一键配置与启动，无需手动填写复杂配置。
- 预置常见场景模板（如：微信助手、定时任务、邮件汇总），选择即用。

**零摩擦上手路径**

- 从安装到首个可用场景的路径尽可能短、步骤尽可能少。
- 结合 Landing 页面、应用内引导、文档教程形成闭环：了解 → 安装 → 体验 → 深入。
