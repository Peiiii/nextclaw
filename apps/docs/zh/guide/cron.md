# 定时任务

定时任务让一条已经跑通的工作按计划再次执行，例如每天生成简报、每周汇总项目、固定时间提醒或周期检查一个页面。

![NextClaw 定时任务列表](/product-screenshots/nextclaw-cron-job-page-cn.png)

## 先手动跑通

在创建计划之前，先在普通会话中执行一次。确认输入来源、工作目录、输出格式和发送目标都正确，再把同一条成熟指令交给 Agent 创建定时任务。

```text
把刚才的简报任务设为每个工作日上午 8:30 执行。读取 daily-sources.md，生成 Markdown 简报，并发送到已配置的飞书群。没有新内容时也要说明检查时间和来源。
```

## 一个定时任务包含什么

- 要执行的指令；
- 时间计划和时区；
- 使用的工作目录或会话；
- 启用状态；
- 上次和下次执行时间；
- 需要发送结果时使用的渠道。

## 管理和检查

在定时任务页面可以查看列表、启用或停用任务、立即运行，并检查上次与下次执行时间。修改模型、目录、渠道或凭证后，应立即手动运行一次，确认旧计划仍然有效。

## 适合和不适合自动化的工作

适合：低风险、输入稳定、结果容易检查、失败可以重试的工作。

谨慎使用：自动发布、批量删除、对外发送敏感内容、修改线上数据或会产生费用的操作。这些任务应保留明确确认步骤。

完整案例见[定时生成并发送简报](/zh/tasks/scheduled-brief)。
