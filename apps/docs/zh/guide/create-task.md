# 创建第一个任务

第一次任务不要只问一个知识问题。选择一件有材料、有结果、十几分钟内能检查的真实工作，例如整理一份表格、汇总几个网页、查看一个项目或生成一个小页面。

## 1. 新建任务

在主界面点击“新任务”。根据任务选择会话类型和工作目录：

- 处理文件、写代码或生成应用时，选择对应项目目录。
- 只做临时调研或写作时，可以使用默认工作区。
- 需要特定运行时或专用 Agent 时，在创建前选择相应入口。

工作目录决定 Agent 可以从哪里读取材料、把结果保存到哪里。不要随意选择包含大量无关或敏感文件的上级目录。

## 2. 把任务说完整

一条容易执行的任务通常包含四部分：

```text
目标：要完成什么
材料：文件、目录、链接或已有内容在哪里
结果：希望得到文档、表格、图片、代码还是页面
边界：时间范围、格式、是否允许修改原文件、完成标准
```

例如：

<div class="nc-task-prompt">
  <p>读取 sales-data 目录里的 CSV，检查缺失值和重复记录，按月份统计各产品线销售额。生成两张图和一份 HTML 报告，写出三条有数据依据的结论。不要修改原文件，把结果保存到 analysis-output。</p>
</div>

## 3. 添加材料

你可以直接说明本地路径、添加文件、粘贴链接或在消息中引用 Panel App。材料多时，先让 Agent 列出它找到的文件和来源，再确认处理范围。

## 4. 发出任务

发送后先观察 Agent 对目标的理解。如果它询问指标口径、输出格式或权限，先回答清楚再继续。发现目录或方向错误时立即停止，不必等它完成。

## 完成标准

第一个任务成功，不是只看到一句回复，而是：

- Agent 使用了正确的材料和目录；
- 你能看懂它做过哪些关键操作；
- 结果能在会话或右侧工作区打开；
- 你可以继续提出修改，而不用重新开始。
