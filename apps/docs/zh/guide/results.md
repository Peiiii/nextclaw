# 查看任务结果

NextClaw 的结果不只是一段聊天回复。一个任务可能同时交付说明、文件、源码、Markdown、图表、HTML 页面、图片或 Panel App。

## 在哪里看结果

### 会话正文

适合查看结论、说明、状态和需要你确认的问题。重要结论应能指向具体文件、来源或数字。

### 会话工作区

右侧工作区可以查看项目文件、当前打开的文件、子任务和与本会话相关的定时任务。生成的新文件也应出现在约定目录中。

### 文件预览

Markdown、代码、HTML、文档、表格和演示文稿可以直接打开。HTML 既可以看源码，也可以看实际渲染结果；代码修改可以结合差异查看。

![在会话旁查看 HTML 数据报告](/product-screenshots/nextclaw-workspace-preview-cn.png)

### Panel App

如果交付物是仪表盘、表单、计算器或交互页面，可以作为 Panel App 留在右侧继续使用，而不是只保存一张截图。

## 如何检查

1. **对目标**：交付物是否回答了最初的问题。
2. **对材料**：使用的文件、链接、数据范围是否正确。
3. **对过程**：是否发生了不需要的覆盖、删除、发送或外部请求。
4. **对结果**：文件能否打开，图表数字能否复算，页面能否使用。
5. **对边界**：来源、限制和不确定项是否说清楚。

## 继续修改

指出具体对象和变化即可：

```text
保留当前数据处理结果，只把报告首页改成管理层摘要，并增加数据更新时间。
```

```text
这个页面在窄屏下表格溢出了，请修复后重新打开预览。
```

修改完成后再次检查最终文件，而不是只根据 Agent 的文字确认。
