# 远程访问

远程访问的目标很明确：把本地 NextClaw UI 变成一套可以从其他设备接入的个人控制台，而且主流程要在界面里完成，而不是依赖命令行经验。

## 这个页面解决什么

`设置 -> 远程访问` 现在把下面四件事放进了一个页面：

- 平台账号登录或注册
- 设备级远程配置
- 托管服务控制
- 远程诊断

这意味着你可以直接在浏览器里完成一条完整闭环：

1. 登录 NextClaw 平台
2. 打开远程访问并设置设备名
3. 启动或重启托管服务
4. 运行诊断并确认连接器健康

## 为什么服务控制也要放进来

远程访问真正生效，依赖的是托管的 NextClaw 后台服务。仅仅保存设置，只是把配置写入文件；只有托管服务启动或重启后，远程连接器才会按新配置运行。

所以 UI 里直接提供了：

- `启动服务`
- `重启服务`
- `停止服务`

如果当前页面本身就是由这个托管服务提供的，那么停止或重启时页面会短暂断开。这是预期行为。

## 你可以确认什么

总览区和诊断区可以帮助你确认：

- 平台 token 是否已经写入
- 远程访问是否已在配置中启用
- 托管服务是否在运行
- 连接器是否已连接
- 本地 UI 健康检查是否可达
