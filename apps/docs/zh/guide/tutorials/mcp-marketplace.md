# MCP 教程（不碰命令行）

这篇教程是给小白用户的。

目标只有一件事：

- 不打开终端
- 不写命令
- 只通过 NextClaw UI 完成一次 MCP 的安装、检查和使用

如果你只想知道结论，可以先记住这 3 步：

1. 去 `Marketplace -> MCP`
2. 点 `Install`
3. 装完先点 `Doctor`，通过后再去聊天里使用

## 1）先理解 MCP 是什么

MCP 可以理解成“给 AI 增加外部工具能力”的一种标准方式。

它不是单独的聊天入口，而是：

- 先安装成一个可用工具源
- 再被聊天里的 Agent 自动发现和调用

所以它的使用顺序永远是：

1. 安装 MCP
2. 确认 MCP 可用
3. 在聊天里正常提需求

不是“装完 MCP 后，再去打开一个单独的 MCP 页面聊天”。

## 2）打开 MCP 市场

在 NextClaw 左侧边栏找到：

- `Marketplace`
- 然后进入 `MCP`

你会看到两部分：

- `MCP 市场`：可安装的 MCP 列表
- `已安装 MCP`：你已经装过的 MCP

每张卡片通常会展示：

- 名称
- 简介
- 支持的传输方式（例如 `STDIO` / `HTTP` / `SSE`）
- 当前状态

## 3）安装一个 MCP

在 MCP 市场里找到你要的条目，点击 `Install`。

安装弹窗里通常会看到这些字段：

- `Server 名称`
- `默认对全部 Agent 可用`
- 某些 MCP 自带的输入项

对小白用户，默认建议是：

- `Server 名称` 不改
- `默认对全部 Agent 可用` 保持开启
- 只有明确知道含义时再修改额外输入项

安装成功后，这个 MCP 会立即出现在：

- `已安装 MCP`

而且通常不需要重启服务。

## 4）一定先点 Doctor

安装成功不等于一定能用。

最关键的一步是：

- 在已安装卡片上点击 `Doctor`

你要重点看这几个结果：

- `Accessible`
- `Transport`
- `Tools`

最理想的状态是：

- `Accessible = true`
- `Tools > 0`

这表示：

- NextClaw 已经能连上这个 MCP
- 并且已经发现了它暴露出来的工具

如果 `Doctor` 没通过，不要急着去聊天里试，因为这时大概率还不可用。

## 5）通过后怎么真正使用

当 `Doctor` 通过后，回到你的聊天页面即可。

重点是：

- 不需要手动“打开 MCP”
- 不需要手动“切到 MCP 模式”

你只要正常向 Agent 提需求。

例如：

- “帮我调用浏览器看一下当前页面”
- “帮我读取这个服务的接口返回”
- “帮我用这个外部工具检查状态”

如果当前 runtime/agent 支持 MCP，它会把已安装 MCP 作为补充工具源使用。

## 6）以 Chrome DevTools MCP 为例

`Chrome DevTools MCP` 是一个很典型的例子。

你可以在 MCP 市场里安装它，但要注意：

- 它不是“安装后 100% 立刻可用”的纯文档型工具
- 它依赖本机 Chrome 处于可连接状态

所以这类 MCP 的正确顺序是：

1. 在 UI 里安装
2. 点 `Doctor`
3. 如果 `Doctor` 失败，再按页面或后续引导排查浏览器侧条件

也就是说：

- 安装动作可以不碰命令行
- 但某些 MCP 本身依赖外部程序状态
- 所以是否真正可用，必须以 `Doctor` 结果为准

## 7）什么时候用 Enable / Disable / Remove

安装完成后，你会在卡片上看到几个常见操作：

- `Disable`
- `Doctor`
- `Remove`

建议这样理解：

- `Disable`
  - 暂时停用
  - 配置还保留
  - 以后可以再启用

- `Doctor`
  - 检查是否真的连通、是否真的能发现工具

- `Remove`
  - 彻底移除这条 MCP 配置
  - 适合装错了、不再需要了、或者准备重装

如果你只是“今天先不用”，优先用 `Disable`。

如果你确定不要了，再用 `Remove`。

## 8）给小白的推荐顺序

第一次使用 MCP，建议按这个顺序来：

1. 先选一个官方或说明最完整的 MCP
2. 安装时保持默认值
3. 安装后立刻点 `Doctor`
4. `Doctor` 通过后，再去聊天里测试
5. 如果失败，先排查该 MCP 的外部依赖，不要直接怀疑聊天页面

## 9）你现在可以怎么开始

如果你是第一次接触 MCP，最推荐的实际动作是：

1. 打开 `Marketplace -> MCP`
2. 先挑一个你最容易理解用途的条目
3. 点击 `Install`
4. 点击 `Doctor`
5. 看到可访问后，再回聊天页测试一次
