# 为什么用 NextClaw

NextClaw 想解决的不是“再给你一个聊天窗口”，而是把你原本散落在不同模型、渠道、工具、网站和本机系统里的能力，逐步收拢成一个统一入口。

它运行在你自己的机器上。你可以先在浏览器里完成配置，再决定它要不要接入聊天渠道、自动化任务、远程访问和更长期的托管方式。

## 它对你真正有用的地方

- 先表达目标，不必先想“该打开哪个软件”
- 可以把模型、渠道、工具和宿主机器连成一条执行链路
- 先从一个真实小任务开始，而不是先学完整系统
- 用熟之后，再把它变成一个长期可用的个人操作层

## 你最先会得到什么

- 一个能在本机跑起来的 AI 控制面
- 一个能接入 Provider 和默认模型的浏览器配置入口
- 一个可以继续扩展到渠道、工具和定时任务的工作底座
- 一条从“先跑通”到“长期可用”的自然路径

## 10 分钟内先跑通

先别管完整命令体系，也别先研究架构。目标只有一个：把它跑起来，并完成一个真实任务。

```bash
npm i -g nextclaw
nextclaw start
```

浏览器打开 `http://127.0.0.1:55667`。
