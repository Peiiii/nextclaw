# NextClaw 是什么

NextClaw 是一个本地优先的 AI Agent 工作台。你给出目标、材料和工作目录，它可以在同一个任务里调用模型、技能、浏览器、终端和本机文件，持续执行，直到交付可以检查的回复、文档、图表、代码或小应用。

普通聊天助手通常停在“告诉你怎么做”。NextClaw 更适合那些需要打开资料、操作文件、运行工具、生成产物并继续修改的任务。

## 一个任务如何完成

1. **描述结果**：说明要完成什么、材料在哪里、结果要保存成什么。
2. **Agent 开始执行**：读取相关文件，按需要使用工具、技能、MCP 或浏览器。
3. **你随时跟进**：查看正在进行的操作，补充要求，停止不合适的步骤。
4. **检查交付物**：在会话和右侧工作区查看回复、文件、源码、Markdown、HTML 或 Panel App。
5. **继续或复用**：要求修改结果，交给专用 Agent，保存成 Skill，或设为定时任务。

## 它适合做什么

- 抓取和整理网页数据，生成图表与结论。
- 批量处理本地文件，提取、分类、重命名或生成目录。
- 调研多个来源，写成带出处的文章、报告或方案。
- 查看并修改代码项目，运行命令并检查页面结果。
- 生成图片、HTML 页面、仪表盘和可重复使用的小工具。
- 按时间生成简报、检查状态，再发送到微信、飞书等渠道。

更多完整示例见[任务案例](/zh/tasks/)。

## NextClaw 的工作表面

- **任务与会话**保存目标、消息、执行过程和后续修改。
- **会话工作区**汇集项目文件、打开的文件、子任务和定时任务。
- **右侧预览**直接查看 Markdown、代码、HTML、文档、表格、演示文稿和差异。
- **Agents**为不同工作保留独立身份、记忆、技能、运行时和主目录。
- **Skills 与 MCP**给 Agent 增加可安装、可复用的做事方法和外部工具。
- **Panel Apps**把一次生成的页面或工具变成会话旁边可以继续使用的小应用。
- **Doc Browser**在全局右侧栏打开网页、文档和多个浏览标签。
- **定时任务与消息渠道**让任务可以在固定时间运行，或从常用聊天工具进入。

## 数据和权限

NextClaw 运行在你控制的设备或服务器上。工作目录、会话、配置和密钥由你的运行环境保存；接入的模型、渠道和外部工具仍会收到你明确发送给它们的数据。执行文件修改、外部请求或高影响操作前，应检查目标目录、授权范围和结果。
