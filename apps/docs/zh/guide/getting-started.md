# 快速开始：完成一次端到端任务

这条路径用十几分钟跑通 NextClaw 的完整工作方式：配置一个模型，选择工作目录，发出真实任务，观察执行，并打开最终文件。

## 1. 打开 NextClaw

桌面版安装后直接启动。使用 npm 时执行：

```bash
nextclaw start
```

然后打开 `http://127.0.0.1:55667`。Docker 用户使用部署时配置的地址。

如果界面无法打开，先运行：

```bash
nextclaw status
nextclaw doctor
```

## 2. 配置一个可用模型

在设置中选择你已有凭证的模型提供方，填写认证信息并选择默认模型。也可以使用本地模型或自定义 OpenAI 兼容地址。

先只配置一条能稳定使用的路径。更多选择见[模型与提供方](/zh/guide/model-selection)。

## 3. 创建一个带结果的任务

新建任务，选择一个测试目录，并在其中准备一份短 Markdown 或文本文件。然后发送：

<div class="nc-task-prompt">
  <p>读取当前目录里的材料，整理出五条要点和三项待办。把结果写成 summary.md，保留原文件。完成后告诉我你读取了哪些文件，并打开最终结果。</p>
</div>

这比问“你好”更能确认模型、文件工具、工作目录和结果预览是否一起正常。

## 4. 观察并调整

确认 Agent 读取的是测试目录。它询问范围或格式时直接回答；如果选择了错误文件，立即停止并修正。你可以在同一会话补充：

```text
待办按负责人、截止时间和状态整理成表格。
```

## 5. 打开实际结果

在右侧工作区找到 `summary.md`，检查内容和源文件是否符合要求。再提出一次修改，确认任务可以沿用当前材料继续，而不是重新开始。

## 快速开始完成标准

- 界面能够稳定打开；
- 一个模型可以回复并调用工具；
- Agent 只访问了你选择的工作目录；
- `summary.md` 确实写入本地并能在右侧打开；
- 继续修改时仍保留当前任务上下文。

接下来可以详细了解[如何创建任务](/zh/guide/create-task)和[如何检查结果](/zh/guide/results)，也可以直接挑一个[任务案例](/zh/tasks/)。
