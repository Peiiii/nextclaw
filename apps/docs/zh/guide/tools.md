# 工具与操作

Agent 通过工具对外部世界采取行动。工具可能读取和修改本地文件、运行终端命令、搜索内容、访问网页、发送消息、创建子任务、使用记忆、安排定时任务或生成图片。

## 常见工具类型

- **文件**：读取、搜索、创建和修改文件。
- **终端**：运行项目命令、脚本、测试和本机程序。
- **网页**：搜索资料、打开页面、提取信息和完成浏览器操作。
- **消息**：通过已配置渠道发送或接收内容。
- **Agent 与会话**：创建子任务，读取其他任务交付的结果。
- **图片与展示**：生成图片，打开文件或在工作区展示结果。
- **定时任务**：保存可重复执行的指令和时间计划。

## 工具、Skill 和 MCP 的区别

- **工具**是 Agent 当前可以直接调用的动作。
- **Skill**描述如何完成一类任务，可以组合多个工具和判断步骤。
- **MCP**把外部服务提供的工具和资源接入 NextClaw。

安装一个 Skill 不一定新增底层工具；接入 MCP 也不等于 Agent 自动知道最佳使用方法。复杂工作通常需要把两者配合起来。

## 操作前的判断

读取通常比修改风险低，预览比批量执行安全。涉及删除、覆盖、发送、发布、购买、账号权限或线上数据时，应要求 Agent 先说明对象和影响范围，并在执行后检查真实结果。

## 找不到需要的工具

先在[Skills 与 MCP](/zh/guide/skills-and-mcp)中查找现成能力。仍然没有时，可以让 NextClaw 为稳定流程编写 Skill，或接入提供相应 API/工具的 MCP 服务。
