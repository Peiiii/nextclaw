# 消息渠道

接入消息渠道后，你可以从日常聊天工具向 NextClaw 发任务、补充材料和接收结果，不必先打开本机工作台。

![NextClaw 消息渠道配置](/product-screenshots/nextclaw-channels-page-cn.png)

## 当前可以接入的入口

界面当前覆盖微信、飞书/Lark、QQ、钉钉、企业微信、Telegram、Discord、Slack、WhatsApp、邮箱和 Webhook 等渠道。不同渠道的登录方式、权限和消息能力不同，以对应配置页为准。

## 适合的工作

- 在手机上发一个明确请求，让本机 Agent 处理文件或项目。
- 把现场照片、文档或消息转成待办和报告。
- 接收定时简报、巡检结果和失败提醒。
- 在团队群里触发固定工作流，再把结果发回原会话。

## 接入步骤

1. 先在本地工作台完成一个真实任务，确认模型和工具正常。
2. 在设置中选择一个常用渠道，按要求登录或填写凭证。
3. 检查渠道连接状态和授权范围。
4. 从目标聊天工具发送一条低风险测试任务。
5. 确认回复、文件和发送对象都正确，再用于日常工作。

## 权限与隐私

渠道收到的消息可能被发送到你配置的模型和工具。群聊中还要考虑谁可以触发 Agent、哪些目录允许访问、结果是否可以回传。不要把高权限本机操作直接暴露给没有身份限制的公开入口。

飞书的具体配置见[飞书教程](/zh/guide/tutorials/feishu)。完整案例见[从微信或飞书接收任务](/zh/tasks/chat-channel-work)。
