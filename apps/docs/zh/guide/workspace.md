# 会话工作区

会话工作区把当前任务涉及的项目、文件和后续工作放在会话旁边。处理本地资料、代码项目或生成应用时，这里通常比聊天正文更接近最终结果。

## 项目文件

选择工作目录后，可以浏览目录中的文件并打开需要查看的内容。Agent 新生成或修改的文件也应落在这个目录或你指定的输出目录中。

支持的预览包括 Markdown、代码、HTML、常见文档、表格和演示文稿。HTML 可以在源码与渲染结果之间切换，代码修改可以查看差异。

![会话旁的文件和 HTML 预览](/product-screenshots/nextclaw-workspace-preview-cn.png)

## 打开的文件

打开的文件会保留在当前工作区，方便你在对话、源码和结果之间切换。让 Agent 修改某个文件时，完成后直接重新打开并检查实际内容。

## 子任务

复杂工作可以拆成子任务。例如一项调研可以分别收集来源、清洗数据和撰写摘要。子任务应有独立目标和交付物，主会话负责汇总与验收。

## 会话定时任务

与当前工作相关的定时任务可以在工作区中查看。周期运行前，先确认它使用的工作目录、指令和发送渠道没有过期。

## 使用建议

- 任务开始前确认目录，不要把整个用户主目录当成默认工作区。
- 批量修改前先要求列出计划和影响文件。
- 约定独立输出目录，保留源文件。
- 结果完成后检查实际文件，不只看聊天摘要。
