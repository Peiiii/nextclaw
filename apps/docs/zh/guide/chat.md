# 任务与会话

在 NextClaw 里，会话不是单纯的聊天记录，而是一项任务持续工作的容器。目标、消息、工具活动、工作目录、文件结果和后续修改都在这里延续。

## 新任务和继续任务

以下情况适合新建任务：

- 开始处理另一个项目或工作目录；
- 新目标与当前材料没有关系；
- 需要改用不同 Agent 或运行时；
- 希望把实验过程与长期工作隔开。

以下情况适合继续原会话：

- 修改刚生成的文件、图表或页面；
- 继续追问同一批资料；
- 根据当前结果增加下一步；
- 需要保留前面已经确认的口径和约束。

## 会话类型

NextClaw 可以承载 Native、Codex、Claude Code、微信、飞书和定时任务等不同运行入口。它们的工具和交互方式可能不同，但都应围绕清楚的目标、工作目录和可检查结果使用。

## 会话列表

会话列表用于返回最近任务。名称应能说明正在做什么，避免长期保留大量“你好”“测试”之类无法辨认的会话。一个项目有多项工作时，可以按结果命名，例如“七月销售分析”“官网文案校对”“发版检查”。

## 长上下文

会话可以保留多轮上下文，也会在内容过长时整理信息。关键路径、最终口径、不可修改范围和交付标准仍应写在当前任务中，避免重要约束只存在于很早的一条消息里。

## 会话内可以继续扩展

- 打开[会话工作区](/zh/guide/workspace)查看项目与文件。
- 创建子任务，让不同 Agent 分别处理部分工作。
- 为当前会话安排定时任务。
- 引用 Panel App 或 Skill 继续工作。

第一次使用请从[创建第一个任务](/zh/guide/create-task)开始。
