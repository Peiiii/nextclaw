# 对话与会话

对话是用户和 NextClaw 交互的表面。会话保存上下文，让多轮任务不需要每次重新说明背景。

## 用户需要理解的概念

- 对话：一次或多次消息交互。
- 会话：承载上下文、模型绑定和历史记录的工作单元。
- 模型绑定：某个会话可以使用特定模型。
- 自动化会话：定时任务可以继续某个会话，也可以使用独立上下文。

## 第一次怎么用

1. 在 UI 新建或打开一个会话。
2. 发起一个真实任务。
3. 连续追问两三轮，确认上下文被保留。
4. 如果任务会长期跟进，再考虑自动化或渠道。

## 什么时候需要多个会话

- 不同项目不要混在一起。
- 不同模型策略可以拆开。
- 临时测试和长期任务要分开。
