# Agent 与子任务

Agent 是一个可以长期保留自身身份、工作目录、记忆、技能和运行时的协作者。你可以为不同类型的工作建立不同 Agent，也可以在复杂任务中让它们分别完成子任务。

![NextClaw Agent 管理界面](/product-screenshots/nextclaw-agents-page-cn.png)

## Agent 可以保留什么

- **身份与角色**：它负责什么，输出应遵循什么要求。
- **主目录**：默认从哪里读取和保存文件。
- **记忆**：与该 Agent 工作相关的持久信息。
- **技能**：经常使用的步骤和工具说明。
- **运行时与上下文配置**：使用 Native、Codex、Claude Code 等不同执行环境。

## 什么时候创建专用 Agent

- 同一类任务经常重复，且目录和标准比较稳定。
- 不同项目需要隔离文件、记忆或技能。
- 开发、写作、调研等任务需要不同运行时。
- 你希望把临时尝试与日常主 Agent 分开。

第一次使用不需要先建很多 Agent。先用默认 Agent 跑通真实任务，再把稳定差异沉淀下来。

## 子任务与协作

复杂任务可以拆成多个有明确交付物的子任务，例如：

1. 一个子任务收集并核对来源；
2. 一个子任务整理数据和图表；
3. 主任务读取这些结果并形成最终报告。

拆分时不要只写“负责研究”这类模糊角色。每个子任务都应说明输入、输出、截止条件和汇总位置。

## 检查结果

主任务仍要负责最终验收。多个 Agent 给出的结论可能冲突，文件也可能写入不同目录；汇总前应检查来源、版本和实际产物。
