# 完成第一个任务之后

第一次跑通之后，不需要立刻配置所有功能。根据刚完成的任务，选择最有价值的一步继续。

## 结果还需要修改

继续留在原会话里，指出要改的对象和验收标准。文件、来源和执行记录都还在，不必从头开始。

## 经常做同一类工作

- 为它创建一个专用 [Agent](/zh/guide/multi-agent)，固定主目录、角色和技能。
- 把稳定步骤整理成 [Skill](/zh/guide/skills-and-mcp)，下次直接复用。
- 如果结果本身是一个页面或工具，把它做成 [Panel App](/zh/guide/panel-apps)。

## 需要定时运行

先手动完成一次并确认结果，再创建[定时任务](/zh/guide/cron)。日报、巡检、提醒和周期汇总都适合从一条已经跑通的指令开始。

## 希望离开电脑也能发任务

接入[消息渠道](/zh/guide/channels)，从微信、飞书、Telegram、Slack 等入口发送请求和接收结果。需要从外部直接打开工作台时，再配置[远程访问](/zh/guide/remote-access)。

## 任务需要更多工具

在技能市场或 MCP 市场查找相应能力。安装前先看它会访问哪些数据、调用哪些命令，再用低风险任务验证。

不知道下一步做什么，可以从[任务案例](/zh/tasks/)挑一个与你日常工作接近的例子。
