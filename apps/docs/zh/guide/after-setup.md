# 第一个有用工作流

快速开始证明 NextClaw 能跑。这个页面证明它值得继续用。

目标是完成一个简单但真实的工作流：配置模型，发起一次有用对话，再创建一个低风险自动化任务。

## 1. 先确认健康状态

```bash
nextclaw status
nextclaw doctor
```

如果这里已经报错，先去 [故障排查](/zh/guide/troubleshooting)。

## 2. 做一次真实对话

打开 UI，发一个你当天真的会用到的请求，例如：

```text
根据下面的零散想法，整理成一份可以发给同事的执行清单。
```

不要先测试玩具问题。第一件事最好和你的真实工作有关。

## 3. 建一个提醒或定时任务

接下来让 NextClaw 主动做一件小事，例如：

```text
每天上午 9:30 提醒我确认今天最重要的一件事。
```

如果你要手动理解自动化能力，看 [运行自动化](/zh/guide/cron)。

## 4. 判断是否需要接入入口

如果你每天都打开某个聊天工具，可以继续接入渠道：

- [接入聊天入口](/zh/guide/channels)

如果你只是先在本机 UI 使用，可以暂时跳过渠道。

## 5. 判断是否需要长期运行

当你准备让 NextClaw 成为日常入口时，继续看：

- [后台运行与自启动](/zh/guide/background-autostart)
- [远程访问](/zh/guide/remote-access)
- [Docker 部署](/zh/guide/tutorials/docker-one-click)

完成到这里，你已经从“能跑”进入“有价值”。
