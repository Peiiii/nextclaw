# 开发一个本地小应用

当一件事会反复做，可以让 NextClaw 把它做成脚本、HTML 页面或 Panel App。例如 CSV 合并器、图片压缩工具、费用计算器、个人看板或资料搜索页。

## 开始前准备

- 先写清这个工具只解决什么问题。
- 准备一两份真实输入样例和期望输出。
- 说明运行环境、是否需要保存数据，以及哪些目录允许修改。
- 第一版只保留核心流程，不要一次加入账号、同步、权限和复杂后台。

## 推荐步骤

### 1. 先确认最小版本

让 NextClaw 复述输入、核心操作、输出和暂不实现的功能。确认后再开始写代码。

### 2. 生成并运行

把代码放在独立目录，说明启动方式。页面或 HTML 结果可以在右侧预览；需要查看源码时切换到对应文件。

![左侧继续与 Agent 修改，右侧直接运行 Panel App](/product-screenshots/nextclaw-panel-app-running-cn.png)

### 3. 用真实样例测试

至少测试正常文件、空文件、字段不一致和重复文件。不要只看页面是否打开。

### 4. 一次改一类问题

先修正确性，再处理布局和细节。描述按钮、输入和具体表现，比只说“不好用”更容易得到稳定修复。

## 结果检查

- 新用户能否不看源码完成核心操作。
- 错误输入是否有明确提示，是否会损坏原文件。
- 页面刷新或重新打开后，必要数据是否按预期保留。
- 项目目录是否包含启动说明和依赖说明。
- 生成结果能否下载或保存到明确位置。

## 继续升级

第一版稳定后，再考虑搜索、批量处理、自动化触发、消息渠道输入或做成可复用 Skill。每次增加一类能力，并保留可运行版本。
