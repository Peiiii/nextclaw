# 任务案例

这里展示的不是功能名称，而是可以直接交给 NextClaw 的具体工作。每个案例都包含要准备的材料、一条可修改的指令、Agent 会经历的关键步骤、最终交付物和检查方法。

## 处理信息与资料

### [抓取数据并生成图表报告](/zh/tasks/data-analysis)

读取网页、CSV 或表格，清洗数据、确认指标，交付图表、结论和 HTML 或 Markdown 报告。

### [整理、重命名并汇总本地文件](/zh/tasks/file-processing)

先盘点目录和预览变更，再分类、重命名、提取内容，并保留可检查的处理记录。

### [调研多个来源并标注出处](/zh/tasks/research-writing)

收集网页和已有资料，比较不同来源，把事实、分歧和引用整理成一份研究文档。

### [根据资料写文章或报告](/zh/tasks/writing)

把采访、笔记、旧稿和参考资料整理成结构清楚的文章、周报、提案或发布说明。

### [汇总用户反馈并排出问题优先级](/zh/tasks/feedback-analysis)

读取工单、群聊记录或问卷，合并重复问题，统计频次，交付带原始证据的优先级清单。

## 创建和持续运行

### [生成图片并保留本地文件](/zh/tasks/image-creation)

从用途、画面和尺寸开始生成图片，把结果保存到本地，再在同一任务中继续挑选或修改。

### [开发一个可在右侧使用的小应用](/zh/tasks/build-local-app)

把重复工作做成 HTML 页面、脚本或 Panel App，并在会话旁预览、调试和继续使用。

### [检查并修改一个代码项目](/zh/tasks/code-project)

让 Agent 先理解仓库、复现问题、修改代码，再运行测试并打开真实结果检查。

### [定时生成并发送简报](/zh/tasks/scheduled-brief)

先手动跑通取数和成稿，再设定执行时间、工作目录和发送渠道。

### [从微信或飞书接收任务](/zh/tasks/chat-channel-work)

从日常聊天入口发送请求，让本机 Agent 继续使用文件和工具，完成后把结果发回来。

## 第一次选哪个

选择一项十几分钟内能验证的真实任务。最好带一份表格、一组文件、几个网页或一个小项目，并明确结果保存在哪里。还没有安装时，先看[安装 NextClaw](/zh/guide/install)和[创建第一个任务](/zh/guide/create-task)。
