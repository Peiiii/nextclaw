# 整理一批本地文件

这个任务适合处理下载目录、项目资料、会议文件、图片、PDF 或一组 Markdown。目标是得到清楚的目录、命名和内容清单，同时保留可检查的变更记录。

## 开始前准备

- 明确要处理的目录，不要一开始授权整个主目录。
- 说明哪些文件只能读取，哪些允许移动、重命名或修改。
- 如果文件重要，先做好备份或使用副本。
- 给出分类规则；没有规则时，先让 NextClaw 提议，不要直接执行。

## 第一次指令：只盘点，不修改

<div class="nc-task-prompt">
  <p>请先扫描这个目录，只读取文件名、类型、大小和必要的内容摘要，不要移动、重命名或删除任何文件。按用途提出分类方案，并生成一份包含原路径、建议新路径和处理理由的 Markdown 清单。</p>
</div>

先检查清单，确认分类规则没有误解，再进入实际处理。

## 第二次指令：按确认结果执行

<div class="nc-task-prompt">
  <p>按刚才确认的清单执行整理。不要删除文件；如果目标文件名已经存在，请跳过并记录冲突。完成后输出实际变更、跳过项和失败项。</p>
</div>

## 推荐步骤

1. 扫描目录并生成盘点结果。
2. 抽查几类文件的内容识别是否准确。
3. 确认目录结构和命名规则。
4. 执行移动、重命名、格式转换或内容提取。
5. 对比处理前后的文件数量和路径。
6. 保留变更清单，方便复核或手动恢复。

## 常见任务

- 按年份、项目或文件类型整理下载目录。
- 批量重命名扫描件和图片。
- 从 PDF、Word 或 Markdown 中提取标题和摘要。
- 合并多份 CSV，或把同类文件转换成统一格式。
- 为一组资料生成目录、索引和缺失项清单。

## 结果检查

- 文件总数是否一致，是否出现意外删除。
- 冲突文件是否被跳过，而不是被覆盖。
- 新文件名是否保留扩展名并可正常打开。
- 处理失败的文件是否列出原因和原路径。
- 变更是否只发生在明确授权的目录中。
