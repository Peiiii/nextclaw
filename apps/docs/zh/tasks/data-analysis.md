# 分析数据并生成图表

这个任务的目标不是只得到一段数据点评，而是交付一份可以检查和继续使用的结果：整理后的数据、图表、结论，以及必要时生成的 HTML 或 Markdown 报告。

## 开始前准备

- CSV、Excel、JSON、网页地址或数据所在目录。
- 你真正想回答的问题，例如销售为什么下降、哪些渠道增长最快。
- 统计范围、时间区间和需要排除的数据。
- 想要的结果格式，例如图表、表格、HTML 报告或 Markdown。

涉及重要数据时，先让 NextClaw 读取和分析副本，不要直接覆盖原文件。

## 示例指令

<div class="nc-task-prompt">
  <p>请读取这份销售数据，检查缺失值和重复记录，按月份统计各产品线的销售额与利润率。生成一张趋势图和一张产品线对比图，再写出三条有数据依据的结论。保留原文件，把整理后的数据和 HTML 报告保存到 analysis-output 目录。</p>
</div>

如果数据来自网页，把网址、需要抓取的字段和时间范围一起说清楚。

## 推荐步骤

### 1. 先理解数据

让 NextClaw 列出字段、数据量、时间范围、缺失值和明显异常。先确认数据能否回答问题，再决定画什么图。

### 2. 确认指标口径

对金额、比例、去重规则和时间粒度做一次确认。相同字段名在不同数据里可能代表不同含义。

### 3. 生成基础结果

先得到整理后的表格、主要统计和少量关键图表。不要一开始生成十几张图，先确认方向正确。

### 4. 在右侧查看报告

HTML、Markdown、代码和本地文件可以在右侧工作区打开。对照图表与源数据，继续要求修改口径、颜色、标签或结论。

## 结果检查

- 图表使用的行数、时间范围和筛选条件是否正确。
- 汇总数字能否从源数据复算出来。
- 图例、单位、时间轴和标题是否清楚。
- 结论是否指向具体数字，而不是泛泛描述。
- 原始文件是否保持不变，输出文件是否放在约定目录。

## 继续完善

- 增加数据来源与更新时间说明。
- 把结论改成适合管理层、客户或技术团队的版本。
- 将固定分析流程做成 Skill，供下次复用。
- 需要周期更新时，先手动跑通，再参考[定时生成并发送简报](/zh/tasks/scheduled-brief)。
