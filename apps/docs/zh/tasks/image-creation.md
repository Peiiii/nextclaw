# 生成图片并保留本地文件

NextClaw 可以根据用途和画面要求生成图片，并把文件留在当前工作目录。你可以在同一任务里查看结果、调整提示、比较版本，再把最终图片用于文章、页面或项目。

![NextClaw 生成图片并返回本地文件](/product-screenshots/nextclaw-image-generation-result-cn.png)

## 准备要求

- 图片用途，例如文章封面、产品示意、海报或社交配图；
- 画面主体、环境、构图和需要避开的元素；
- 横竖比例、分辨率和文件格式；
- 品牌颜色、参考图片和文字留白位置；
- 结果保存目录。

## 示例指令

<div class="nc-task-prompt">
  <p>为一篇介绍本地 AI 工作台的文章生成一张 16:9 封面图。画面是明亮书桌上的电脑和几份正在整理的文档，界面内容不要出现可读品牌文字，整体自然、清晰、适合浅色网页。生成两个方向，把文件保存到 article-assets，并分别说明构图差异。</p>
</div>

## Agent 会做什么

1. 把用途和画面要求整理成生成提示。
2. 调用可用的图片生成能力。
3. 把结果保存到约定目录并在会话中展示。
4. 根据你的反馈调整构图、光线、主体或比例。

## 检查结果

- 主体和用途是否一致，而不是只有氛围。
- 尺寸、比例和清晰度是否符合使用位置。
- 人物、文字、手部和细节是否出现明显错误。
- 是否包含不应出现的商标、隐私信息或受限制素材。
- 最终文件是否真的存在于约定目录。

图片确定后，可以继续让 Agent 压缩、命名并写入文章或应用资源目录。
