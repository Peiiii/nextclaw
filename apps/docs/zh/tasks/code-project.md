# 检查并修改一个代码项目

把代码项目作为工作目录后，NextClaw 可以先阅读现状、复现问题、修改实现、运行测试，再打开实际页面或结果验收。这个流程适合修复缺陷、实现小功能和维护脚本。

## 准备信息

- 项目目录和运行方式；
- 期望行为、当前问题和复现步骤；
- 不能触碰的模块或未提交改动；
- 必须运行的测试、类型检查和浏览器验收；
- 是否允许安装依赖、访问网络或修改配置。

## 示例指令

## Agent 会做什么

1. 读取项目说明、工作区状态和相关源码。
2. 按复现步骤看到真实问题，确定 owner。
3. 进行范围清楚的修改，不覆盖已有用户变更。
4. 运行与改动相关的测试、类型检查和构建。
5. 对用户可见问题执行浏览器或运行时验收。

## 检查结果

- 改动是否解决真实复现，而不是只让测试变绿。
- 是否修改了无关文件或覆盖未提交内容。
- 是否引入重复实现、临时兼容分支或新警告。
- 类型、测试、构建和实际界面是否都符合要求。
- Agent 声称完成的命令是否有可查看的结果。

稳定的仓库规范和验证步骤可以交给专用 Agent 或 Skill；一次性任务仍应保留明确范围。
