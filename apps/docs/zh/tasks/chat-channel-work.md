# 从微信或飞书接收任务

接入消息渠道后，你可以从常用聊天入口发起任务、补充文件和继续追问，NextClaw 仍然使用运行它的电脑或服务器上的模型、文件、工具和会话。

## 适合的任务

- 在群里收到文件后，整理重点和待办。
- 离开电脑时继续查看任务进度并补充要求。
- 把定时简报、检查结果或完成通知发回指定会话。
- 让团队从固定入口提交格式明确的请求。

## 开始前准备

1. 先在本地界面完成[快速开始](/zh/guide/getting-started)。
2. 选择一个你真正会使用的渠道，不要第一次同时接入多个平台。
3. 按[接入消息渠道](/zh/guide/channels)或对应教程完成账号、权限和连接配置。
4. 确保 NextClaw 在接收消息时保持运行。

## 第一次测试

先发送一条低风险、结果容易检查的消息：

<div class="nc-task-prompt">
  <p>请回复当前时间，并告诉我你正在使用哪个会话。不要读取或修改任何本地文件。</p>
</div>

确认消息能进入正确会话、回复能返回原渠道，再测试图片、文件和本机任务。

## 带文件完成一个任务

<div class="nc-task-prompt">

</div>

## 结果检查

- 消息是否进入预期的 Agent 和会话。
- 群聊和私聊的触发范围是否符合设置。
- 文件、图片和长消息是否能完整接收。
- 回复是否泄露了不应发回群聊的本地路径、密钥或私人内容。
- 电脑休眠、服务退出或网络中断时，是否有可理解的失败表现。

## 安全建议

- 为群聊设置更严格的触发和权限范围。
- 涉及命令执行、文件修改和外部发送时保留确认步骤。
- 不要把访问密钥直接发在聊天消息里。
- 为长期任务使用独立会话，避免不同群聊共享不相关上下文。
