# 根据资料写文章或报告

这个案例适合把采访、会议记录、旧文档、零散笔记和参考链接整理成一份可以继续编辑的文章、周报、提案或发布说明。目标不是一次生成漂亮文字，而是让结构、事实和语气都能检查。

## 准备材料

- 必须包含的事实、数字和引用；
- 旧稿、笔记、录音整理或参考文档；
- 读者、用途和发布渠道；
- 字数、结构、语气和截止时间；
- 不能公开或仍需确认的信息。

## 示例指令

<div class="nc-task-prompt">
  <p>读取 launch-notes 目录里的会议记录、功能清单和旧版发布说明，写一篇面向现有用户的版本介绍。先列出事实清单和文章大纲，再写 1200 字左右的中文初稿。不要写内部决策过程，不要编造效果数据。把事实核对表和文章分别保存为 Markdown。</p>
</div>

## Agent 会做什么

1. 盘点材料，区分已确认事实、观点和缺失信息。
2. 根据读者和用途提出文章结构。
3. 先生成可核对的大纲，再展开正文。
4. 把不确定内容标出来，不用流畅文案掩盖事实缺口。
5. 保存初稿，供你在右侧打开和继续修改。

## 检查结果

- 每个功能、数字和时间是否能在材料中找到依据。
- 开头是否直接说明读者能得到什么。
- 是否混入“我们如何讨论和设计”之类内部过程。
- 段落是否自然，中文是否有翻译腔或抽象词堆叠。
- 最终文件是否保留来源和待确认项。

## 继续使用

让 Agent 按官网、邮件、社交平台或内部周报分别改写，而不是用一份稿子硬套所有渠道。稳定的品牌语气、事实核对和发布检查可以整理成 Skill。
