# 2026-06-03 · 实时 Agent 进展：现状与下一步

发布时间：2026-06-03  
标签：`产品` `实时状态` `Agent` `会话`

## 摘要

NextClaw 当前已经具备一组和实时 Agent 进展相关的底层能力：

- 会话列表可以随着会话变化实时更新。
- 会话可以显示 `running / idle` 等运行状态。
- 会话摘要可以增量 upsert / delete。
- NCP 事件可以表达运行生命周期、推理、工具调用、工具结果和上下文窗口变化。

这些能力还没有被完整组织成“任务进展面板”。当前更准确的结论是：**实时任务可观察性的底座已经存在，产品层还需要进一步投影和整理。**

## 当前结果

| 能力 | 当前状态 | 用户可见结果 |
| --- | --- | --- |
| 会话实时更新 | 已具备 | 会话列表可跟随 session 变化更新 |
| 运行状态 | 已具备基础形态 | 运行中的会话可显示为 `running` |
| 会话摘要增量更新 | 已具备 | 新会话、更新会话、删除会话可同步到列表 |
| NCP 过程事件 | 已具备协议基础 | 可表达 run、reasoning、tool call、tool result 等过程 |

## 能力边界

当前还不是完整的全局任务驾驶舱。

主要缺口：

- 列表层主要展示会话级状态，还没有展示完整任务阶段。
- `running / idle` 颗粒度较粗。
- NCP 过程事件还需要被聚合成用户能直接理解的摘要。
- 后台任务、定时任务、子 Agent 任务还没有统一进展视图。

## 下一步

可以优先做三件事：

1. 把会话列表的运行状态扩展为“最近阶段 / 最近事件 / 是否等待用户”。
2. 把 NCP 事件投影成任务进展摘要，而不是只在对话流里展示。
3. 把会话、子会话、定时任务和后台任务统一到同一类进展模型里。

这篇文章只记录当前事实和下一步方向，不把尚未完成的产品形态写成已经完成。
