# 2026-07-16 · NextClaw v0.24.0

发布时间：2026-07-16
标签：`release` `npm` `minor`

NextClaw v0.24.0 是一次 npm stable minor 更新。本版本把“项目”从会话附属信息推进成更独立的工作对象：你可以先创建项目、选择项目目录，再从项目上下文里开启任务；聊天输入框也能直接引用当前项目里的文件和目录，让 Agent 更容易围绕真实材料工作。

![NextClaw 新建项目时选择服务端目录](/release-notes/nextclaw-v0.24.0-project-workflow.png)

## 功能

- 新增独立项目注册表与项目模板。项目即使还没有会话，也会保留在项目列表中，适合先搭好知识库、产品资料或代码工作区。
- 新建项目时可以选择空项目或知识库模板，也可以浏览运行 NextClaw 服务的那台机器上的目标目录。
- 服务端目录选择器支持常用位置、地址导航、搜索、新建文件夹、单击选择和双击进入，让项目目录选择更接近本机文件管理体验。
- 聊天输入框支持通过 `@` 搜索并引用当前项目中的文件或目录，发送时会在项目边界内补充受限、可控的上下文。

## 增强

- 项目视图现在能展示没有会话的项目，并从项目分组直接开启新任务。
- 运行时更新检查、下载和切换状态更连贯，完成更新后会自动切换到新的运行包。
- 当前版本和可用更新的更新说明链接更容易从版本号区域进入。
- Markdown 标题间距、文件预览行号滚动和 provider 嵌套模型 ID 处理更加稳定。

## 修复

- 中文输入法候选词确认和取消时，聊天输入框的光标位置更稳定。
- 会话模型恢复、重试错误提示、技能选择和可折叠会话列表图标在多种交互状态下更一致。
- 更新流程不再因为页面状态停滞而让用户误以为仍停在旧版本。

## 默认行为与兼容性

- 本版本不需要手动迁移。
- NPM 安装用户可以继续通过 stable 更新链路，或重新安装 `nextclaw@latest` 获取本版本。
- 项目目录仍然是运行 NextClaw 服务的机器上的路径；默认工作区仍保留为默认位置，不会被强制注册成项目。
