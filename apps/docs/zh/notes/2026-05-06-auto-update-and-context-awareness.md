# 2026-05-06 · 自动更新与长会话上下文管理上线

发布时间：2026-05-06  
标签：`release` `updates` `context` `chat`

NextClaw 最近补上了几项更偏“长期使用体验”的能力：自动更新、上下文窗口统计，以及长会话自动压缩。

它们不是新的聊天噱头，而是让 NextClaw 更像一个可以长期运行、持续工作的个人操作层。

## 你会感受到什么

- NextClaw 可以更顺滑地跟随新版发布，减少手动更新和重新安装的负担。
- 聊天输入区会显示当前会话预计占用了多少上下文窗口。
- 鼠标悬停在上下文圆环上，可以看到预计占用、总窗口、剩余空间等细节。
- 长会话接近上下文上限时，NextClaw 会自动压缩较早上下文。
- 压缩记录会出现在消息流里的正确位置，而不是把旧消息直接删掉。
- 原始历史仍然保留；压缩只影响后续送给模型的上下文组织方式。

## 为什么做这次更新

NextClaw 的目标不是只完成一轮问答，而是支撑持续的工作流。长会话、反复追问、多步骤任务和长期项目都会遇到同一个问题：上下文会增长，用户也需要知道系统正在怎么管理这些上下文。

这次更新让上下文管理变得可见、可解释、可继续。你能看到当前窗口占用，也能看到系统什么时候把较早历史压缩成摘要，从而让后续模型请求继续带着关键背景往前走。

自动更新则解决另一个长期使用问题：NextClaw 应该尽量把能力升级带到用户面前，而不是让每次更新都变成一次手工维护。

## 怎么体验

更新到包含本次能力的 NextClaw 版本后，打开任意聊天会话即可看到输入区旁的上下文窗口圆环。

如果你把某个 agent 的上下文窗口配置得较小，或者在同一会话里持续进行较长任务，就更容易看到自动压缩记录出现在消息流中。
