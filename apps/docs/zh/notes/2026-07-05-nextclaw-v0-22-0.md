# 2026-07-05 · NextClaw v0.22.0

发布时间：2026-07-05
标签：`release` `feature` `enhancement` `fix`

NextClaw v0.22.0 是一次 NPM stable minor 更新。本期重点是让用户在更新前能清楚看到：新增了哪些能力、哪些体验被增强、哪些问题被修复。

## 功能

- Agent 运行中可以继续排队输入，当前任务结束后会按顺序发送。
- `show_file`、`show_url` 和 `show_panel_app` 现在有更明确的展示入口。
- 本地 HTML 可以在右侧工作区里以渲染预览打开，脚本和相对资源更接近真实浏览器行为。
- 新增 `view_image` agent 工具，模型可以读取本地 PNG、JPEG、WebP 和 GIF 图片。

## 增强

- 长会话里可以用浮动按钮快速回到最新消息，已完成的推理和工具过程会折叠成摘要。
- `show_url` 会以浏览器式标签页打开，带地址、刷新和外部打开入口；同一个文件的源码预览和渲染预览也可以同时保留。
- 文件预览支持语法高亮，面包屑可以直接浏览附近目录。
- 新增 Night theme，并改进 Markdown、状态区、错误提示、开关、附件 token、marketplace 详情和 session metadata 的深色可读性。
- SideDock、DocBrowser、自动收起 sidebar、compact tab 和 Slash 面板的交互细节更稳定一致。

## 修复

- 修复上传文件、上传图片或粘贴图片后，发送时附件丢失的问题。
- 图片附件不再默认带 `detail: "auto"`，避免部分 OpenAI-compatible provider 拒绝请求。
- 失败会话的错误信息会显示在聊天详情页，不只停留在会话列表。
- 工具调用中的取消信号会继续传递到 NCP/MCP 工具执行层。
- 修正 session activity 预览中的误报失败状态，减少不必要的错误提示。

## 默认行为与兼容性

- 新配置下 learning loop 默认关闭；已经显式配置过的用户不受影响。
- NCP 事件增加标准 timing 和 message lifecycle 字段，过程耗时和完成状态更可追踪。
- 这是 NPM stable minor 版本，可通过正常的 NextClaw NPM 安装或更新流程使用。

## 升级说明

本版本不需要额外手动迁移。若你的运行环境接入 update manifest，manifest 中的 `releaseNotesUrl` 可以指向本页面；结构化版本说明也会发布为可拉取的 JSON。
