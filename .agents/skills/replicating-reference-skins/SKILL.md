# 参考皮肤复刻验收

## 核心原则

把参考图视为整套视觉系统，不是壁纸。目标不是逐像素临摹截图，而是让目标产品像原生采用了这套设计语言。只有应用壳、内容组件、状态和主视觉同时成立，才算皮肤完成。

“全覆盖”不是让每个页面各写一套补丁，而是先覆盖产品的共享视觉原语，再补页面独有结构。任何未检查的组件都视为未覆盖；不得用“主页面看起来差不多”替代全产品覆盖。

## 底层模型

按以下顺序建设外置皮肤运行时；不得从人物背景直接跳到局部 CSS：

1. 设计 token：颜色、字体、密度、圆角、边框、阴影、动效和透明材质。
2. 语义表面：为应用壳、导航、工具栏、内容区、列表、卡片、输入、弹层和快捷栏建立稳定角色；动态页面需要重新识别新节点。优先使用语义属性和可访问性合同，类名只作为受控补充。
3. 组件配方：同一角色的 default/hover/focus/active/disabled/selected 状态共享一套视觉合同。
4. 页面适配：主题保持统一，但首页、数据列表、会话页和设置页允许不同的信息密度与主视觉强度。
5. 独立资产：纯背景、人物、纹样、签名和贴纸分层渲染；装饰不得烘焙假 UI，也不得遮挡真实内容。

在不修改产品源码的约束下，这五层应由单一注入 owner 完成，不得分裂成多套互相覆盖的补丁。

## 强制工作流

1. 固定参考：记录来源版本、原图和目标视口；区分可直接使用的运行时素材与只表达方向的概念图。
2. 先选验收页：主验收必须使用有真实数据的页面。用户要求本地真实数据时，开发实例默认直接复用真实数据 home、只隔离运行态；禁止复制少量会话冒充真实数据。优先选择列表、筛选器、卡片和滚动区同时存在的页面；空首页只能做补充验收。
3. 先给预览：在首次视觉调整前提供稳定、可点击的本地预览链接。后续保持 URL 不变，每轮改动后刷新实例。
4. 建立覆盖台账：先盘点共享组件、页面专属组件和交互状态，逐项记录 selector/role、页面证据、通过/失败/不适用；没有台账不得开始宣称“全覆盖”。
5. 建立分区矩阵：逐项覆盖左侧栏、顶部栏、主内容背景、列表/卡片、表单/输入、弹层、右侧快捷栏和滚动状态。
6. 提取系统：从参考图明确 token、语义表面、组件状态、页面强度和独立资产；先完成共享运行时，再增加主题特有装饰。
7. 先做共享原语：排版、图标、分隔线、按钮、链接、标签、卡片、表单、弹层、滚动条必须统一；之后才做会话、市场、设置等页面配方。
8. 真实浏览器验收：用同一视口并排比较参考图与实机截图；每个分区和组件族标记通过或失败。高显著度分区未通过时继续调整，不能用编译、单测或空状态截图代替。
9. 补充状态：至少检查有数据、空状态、loading、error、disabled、selected、hover、focus-visible、滚动到底部、窄窗口和无横向溢出。

## 全组件覆盖台账

以下每一族必须覆盖，或以页面证据明确标记不适用：

| 组件族 | 必查组件 |
|---|---|
| 基础原语 | 正文、标题、辅助文案、链接、图标、分隔线、徽标、头像、快捷键提示 |
| 操作控件 | 主/次/危险/图标按钮、分段控件、标签页、下拉触发器、分页与悬浮操作 |
| 表单控件 | input、textarea、select、checkbox、radio、switch、slider、搜索、上传与校验提示 |
| 数据表面 | 卡片、列表行、表格、树、筛选器、统计块、骨架、空状态、错误状态 |
| 导航结构 | 品牌区、主导航、导航分组、会话列表、面包屑、顶部栏、右侧快捷栏、移动导航 |
| 会话内容 | 用户/助手/系统消息、头像、Markdown、链接、引用、表格、代码块、工具过程、附件、流式、失败/重试、输入器与模型/技能选择 |
| 弹层反馈 | dialog、sheet、dropdown、popover、tooltip、command、toast、确认与遮罩 |
| 页面配方 | 首页、真实会话、技能市场、设置/表单、数据列表/详情；装饰强度随页面调整 |

覆盖运行时必须按本轮目标集合差量清理旧标记，仍然有效的角色原地保留，支持 SPA 跳转、懒加载和弹层挂载；禁止每轮全删再加，因为这会重启动画、破坏焦点并制造状态抖动。过宽选择器不得把普通 token 当 tab、把会话工具按钮当卡片主操作，或把所有 article 强行套成同一种卡片。

## 状态合同

每个可交互组件至少定义 `default / hover / focus-visible / active / selected / disabled`；异步组件另查 `loading / success / error`。所有状态必须同时满足：文字可读、焦点可见、布局不跳动、动效尊重 reduced-motion、浅深色资产不冲突。

## 画布与微型动效合同

- 整体背景必须归应用壳或页面共同父容器拥有。Header、内容区、滚动区与输入器应浮在同一画布上；禁止把人物或主视觉默认限制在右上角固定宽高的小矩形，再把其余区域留成无关底色。
- 人物层的运行态矩形应与目标 page canvas 同尺寸，或有明确的艺术构图理由；必须检查顶部、首屏消息之后、输入器后方和底部淡出，不得只截取最容易好看的右上区域。
- 阅读保护应由渐变、mask、透明度和内容表面共同完成，不能通过缩小主视觉回避遮挡。人物焦点要避开标题和主要正文，但视觉气氛仍需贯穿页面。
- 小尺寸加载图标必须以实际 CSS 像素验收，不能只看 SVG 放大图。圆环、尾势或火焰等关键识别特征在 1× 截图中仍须可见；细于一个设备像素的装饰不计为有效细节。
- 循环动效至少连续采样超过一个完整周期，确认 transform/进度持续变化且 DOM 角色身份不变。只看起始帧、GIF 预览或单元测试不能证明动画不会被 hydration/observer 周期性重启。

## 分区验收表

| 分区 | 必查内容 |
|---|---|
| 应用壳 | 画布、整体层次、左右栏关系、背景连续性 |
| 左侧栏 | 品牌区、导航、选中/悬浮、分组、底部账户区 |
| 顶部栏 | 标题、操作区、分隔、透明度、主题装饰 |
| 主内容 | 数据密度、标题层级、滚动、留白与主视觉避让 |
| 卡片/列表 | 行高、图标容器、边框、圆角、阴影、交互态 |
| 输入/筛选 | 输入框、按钮、placeholder、focus、disabled |
| 右侧栏 | 快捷操作、边界、层级、窄屏隐藏策略 |
| 装饰层 | 人物、签名、纹样、贴纸；不得遮住真实内容 |

## 页面巡检门

- 真实会话页：必须能看到多条历史、长文本、代码块、工具过程、用户/助手气泡和输入器。
- 技能市场或数据列表：必须覆盖搜索、筛选、标签页、卡片、按钮、滚动加载、空/错状态。
- 设置页：必须覆盖表单控件、分组、说明、保存/危险操作和弹层。
- 首页：只用于检查主视觉、人物资产和主题识别度，不能替代前三类页面。
- 窄屏：检查侧栏收纳、弹层边界、内容宽度、装饰隐藏和触控态。

## 禁止项

- 禁止只换背景或只调主视觉后声称复刻完成。
- 禁止把带完整 UI 的概念截图整张铺到真实 UI 下方。
- 禁止用无数据页面作为唯一视觉证据。
- 禁止只发截图而不提供可访问的实时预览地址。
- 禁止因某个局部看起来好看而跳过侧边栏、滚动区和交互状态。
- 禁止没有组件覆盖台账就声称整体完成。
- 禁止用宽泛 DOM 选择器制造“覆盖很多”的假象；错配组件也属于未通过。
- 禁止用局部插画窗口冒充整页背景覆盖，也禁止用放大后的图标细节替代实际尺寸验收。

## 完成门槛

最终汇报必须附：参考图、稳定预览链接、有数据页截图、分区验收结果、组件覆盖台账、状态验收、实际尺寸微型动效证据和仍存在的视觉差异。共享组件族、真实会话、市场/列表、设置页任一未检查，背景画布没有覆盖完整页面，循环动效未跨过完整周期，或存在未披露的高显著度差异时，不得使用“全覆盖”“对齐”“复刻完成”或同义结论。
