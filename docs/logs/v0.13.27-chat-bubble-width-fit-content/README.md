# v0.13.27-chat-bubble-width-fit-content

## 迭代完成说明（改了什么）
- 修复聊天消息气泡“短文本仍占用很宽区域”的样式问题。
- 将消息项容器从固定最小宽度 `min-w-[280px]` 改为内容自适应 `w-fit`，保留 `max-w-[92%]` 上限。
- 变更位置：`packages/nextclaw-ui/src/components/chat/ChatThread.tsx`。

- UI 冒烟：

## 用户/产品视角的验收步骤
1. 打开聊天页，发送 2-4 个字的短消息。
2. 观察用户气泡宽度应明显收窄，并与文本长度接近。
3. 再发送长消息，确认长文本仍受 `max-w` 限制并正常换行。
