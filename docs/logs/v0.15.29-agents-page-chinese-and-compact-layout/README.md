# v0.15.29 Agents 页面中文化与紧凑布局优化

## 迭代完成说明

- 将 `agents` 页面从偏展示型的大块 Hero + 独立统计卡结构，收敛为更适合高频管理的紧凑布局：
  - Hero 区压缩纵向高度，并把总数/系统主 Agent/专职 Agent 统计并入 Hero 内联信息区。
  - Agent 卡片缩短彩色头图区、缩小头像尺寸、压缩留白和按钮高度，提升一屏信息密度。
  - 空状态与加载态同步收紧尺寸，避免页面上半屏被大面积占满。
- 对卡片视觉继续做减法，进一步靠近简约管理界面：
  - 去掉 avatar 外层白色包裹框，直接使用单层头像。
  - 收轻卡片阴影和 hover 效果，减少装饰性浮起感。
  - 将主目录区域从独立大圆角信息盒收敛为更轻的分隔式信息区。
- 将 Agent 卡片进一步收敛到与项目现有管理页一致的视觉语法：
  - 去掉大面积彩色头图，改为细条强调色。
  - 统一为白底、灰边、轻阴影的标准卡片形态。
  - 标签收敛到名称行内，并移除自定义 Agent 的“专职”标签，只保留系统主 Agent 标记。
  - 主目录恢复为项目里更常见的灰底信息区表达，降低样式跳脱感。
- 继续降低主目录信息区权重：
  - 去掉主目录灰底信息盒，改为分隔线下的一条轻量元信息。
  - 主目录文本颜色同步减弱，避免与名称、摘要、主操作形成竞争。
- 继续降低 Hero 区统计信息噪音：
  - 去掉“系统主 Agent / 专职 Agent”分拆统计，只保留一个总数。
  - 避免在已能从卡片结构直接感知分类的前提下重复展示汇总信息。
- 调整卡片纵向节奏：
  - 将“主目录 + 操作按钮”作为底部区整体贴底。
  - 中间留白改为由描述区与底部区之间自然伸缩，避免主目录悬在卡片中段。
- 补齐 `agents` 页面中文文案，让中文环境下不再暴露 `Agent Gallery`、`Home Directory`、`Agent identity` 等未本地化表达。
- 新增 `AgentsPage` 页面级渲染冒烟测试，覆盖中文渲染和核心操作按钮可见性。

## 用户/产品视角的验收步骤

1. 在中文环境打开 `/agents` 页面，确认页面主标题、Hero 标签、主目录标签、开始对话按钮等均显示为中文，不再出现 `Agent Gallery`、`Home Directory` 等英文残留。
2. 观察页面首屏，确认统计信息已收进 Hero 区，页面上半屏较之前更紧凑，首屏能看到更多 Agent 卡片内容。
3. 检查单个 Agent 卡片，确认已不再使用大面积彩色头图，而是更克制的细条强调色。
4. 确认头像已不再套额外白色外框，标签被收进名称行内，整体更简约。
5. 确认 Hero 区仅保留一个总数统计，不再重复展示系统主 / 专职拆分数。
6. 确认卡片为白底、灰边、轻阴影，主目录已从独立信息盒降为分隔线下的轻量元信息。
7. 确认主目录与按钮整体贴近卡片底部，卡片中部空白主要出现在描述与底部信息之间，而不是让主目录悬空。
6. 检查头像、摘要、主目录与操作按钮的布局仍清晰可辨，且一屏能看到更多有效内容。
7. 点击“开始对话”，确认仍可按既有逻辑进入对话入口。
8. 打开“新增 Agent”弹窗，确认中文文案与主目录字段提示保持一致。
