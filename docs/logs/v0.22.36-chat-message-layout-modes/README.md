# v0.22.36-chat-message-layout-modes

## 迭代完成说明

本批为聊天消息新增可持久化的“卡片式 / 平铺式”外观配置。卡片式保持既有默认行为；平铺式下，用户消息继续使用右侧气泡，assistant / tool 只在独立头部展示紧凑头像与名称，正文使用完整阅读宽度，时间和复制操作仍留在正文下方。

同时完成了同一消息展示链路中的必要视觉收口：平铺消息轨道限制为 `52rem`，默认输入面板使用同中心线的 `54rem` 轨道；消息图片按自然比例显示并限制为 `32rem`，连续图片保持约 `12px` 间距，统一恢复 hover 放大与灯箱；同段 Markdown 图片与同一消息中的连续图片附件都支持三张成行；图片只保留 `12px` 小圆角，计算样式无边框、无阴影、无底色；Markdown 标题自身提供相对字号 `0.5em` 的下方留白；reasoning/tool/process 与正文保持相同字号和行高，只使用更浅一档的中性灰建立层级。

实现保持单一消息内容链路：布局只切换消息外壳，不复制 Markdown、工具过程、附件或复制动作。图片预览与灯箱从历史附件文件抽为共享展示 owner，由 Markdown 图片和文件型图片共同复用。

## 用户/产品视角的验收步骤

1. 打开“设置 → 外观”，在“卡片式 / 平铺式”之间切换，刷新页面后确认选择仍然保留。
2. 在平铺式会话中确认 assistant 头部只有较小头像和名称；正文从下一行完整铺开，头像下方没有空白占位列；时间和复制操作仍在正文下方。
3. 确认用户消息仍为右侧气泡，切回卡片式后原有消息结构不变。
4. 在桌面端确认会话内容居中且输入面板略宽于会话区；缩窄窗口后确认二者同步占满可用宽度且没有横向滚动。
5. 查看多张连续图片，确认图片不会撑满宽会话、图片之间有稳定间距、只有小圆角而没有边框/底色/阴影；同段至少三张且没有换行的 Markdown 图片、或同一消息内连续三张附件都使用三列，内容边界会中断分组；hover 后放大按钮可见，点击后可打开灯箱并用 Esc 关闭。
6. 查看包含标题、段落、列表、引用、代码块和图片的 Markdown 回复，确认标题下方都有稳定的必要留白。
7. 对比普通回答正文与“已思考 / 已执行 / 运行命令”等提示，确认字号和行高一致，仅文字灰阶不同，且全程只使用黑白灰。
