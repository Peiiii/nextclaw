# v0.13.54-config-meta-architecture-reuse-decouple

## 迭代完成说明（改了什么）
- 更新 `AGENTS.md` 既有规则 `best-solution-over-cost`：明确“简单优先 + 必要抽象 + 通用复用解耦扩展”的落地约束。
- 更新 `AGENTS.md` 既有规则 `avoid-stack-bloat-and-unnecessary-legacy-compat`：新增“禁止主链路堆积无明确触发条件的兜底代码”，并要求兜底具备触发/退出条件。
- 新增规则 `stateless-utility-first`：要求工具函数默认无状态、输入输出可预测，且尽量避免函数作为工具函数入参。
- 新增规则 `root-cause-fix-over-band-aid`：要求问题修复以根因和架构优化为先，禁止仅做表层补丁结束。

## 测试/验证/验收方式
- 结构校验：确认新增/修改规则均位于 `Rulebook` 区域，且严格包含模板字段（约束/示例/反例/执行方式/维护责任人）。
- 语义校验：逐条检查新增内容与 `/config-meta` 诉求一致（通用复用解耦扩展、必要抽象、减少兼容兜底、无状态工具函数、根因修复）。
- 不适用说明：本次仅文档规则变更，`build`/`lint`/`tsc` 不适用（未触达代码构建与运行链路）。

## 发布/部署方式
- 本次为仓库内规则文档更新，无需部署。
- 若需生效到协作流程，按常规代码评审合入主干即可。

## 用户/产品视角的验收步骤
1. 打开 `AGENTS.md`，在 `Rulebook` 中确认已能直接看到四类约束：简单优先与必要抽象、兼容与兜底收敛、无状态工具函数、根因修复优先。
2. 随机抽取一个后续开发任务，检查执行说明是否可直接指导实现决策（是否抽象、是否保留兜底、是否需要架构修正）。
3. 确认规则语言可执行且可审计（包含触发条件、反例、过渡方案与退出条件）。
