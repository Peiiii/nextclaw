# v0.13.34-chat-scroll-logic-move-to-component

## 迭代完成说明（改了什么）
- 将聊天线程滚动逻辑从 `ChatThreadManager` 下沉到 `ChatConversationPanel`，按“纯交互放组件”的原则收敛。
- 组件内新增滚动控制：
  - 会话切换后首屏一次性 `instant` 滚动到底部（等待历史加载完成且存在消息）。
  - 用户手动上滑时不强制打断。
  - 非首屏阶段在消息增长时自动跟随到底部（仅在未上滑状态）。
- `ChatThreadManager` 删除滚动状态与滚动方法，只保留业务动作（删会话、创建会话、跳转）。

## 用户/产品视角的验收步骤
1. 进入任意有历史消息的会话，确认首屏直接展示底部最新消息。
2. 切换到另一个会话，重复验证首屏滚底稳定生效。
3. 在会话里手动上滑后等待新消息，确认不会被自动拉回底部。
4. 保持在底部发送新消息，确认仍会自动跟随到底部。
