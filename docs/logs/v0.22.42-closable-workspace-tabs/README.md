# v0.22.42 会话工作台标签页关闭能力

## 迭代完成说明

- 此前只有文件标签页具备关闭动作；子会话、草稿以及子会话列表、定时任务、项目文件三个稳定页面没有统一的关闭语义，其中稳定页面还会被 view model 固定投影出来。
- 现在由 `ChatThreadManager` 统一承接工作台标签页关闭动作：概览保持固定，其他标签页均可关闭；关闭当前标签页会恢复历史中的可用页面，没有可恢复项时回到概览。
- store 只保存当前工作台已关闭的标签页条目，并通过既有 Zustand persist 链路恢复；重新从概览或列表进入对应页面时，manager 会清除该关闭记录，不新增组件本地状态或第二套持久化路径。

## 用户/产品视角的验收步骤

1. 打开任意会话的会话工作台，确认概览标签页没有关闭入口。
2. 分别关闭子会话、定时任务、项目文件、具体子会话、草稿或文件标签页，确认对应标签页消失且工作台保持打开。
3. 关闭当前标签页，确认自动回到最近可用标签页；无其他可用项时回到概览。
4. 从概览重新进入已关闭页面，确认对应标签页重新出现；刷新后确认当前关闭状态保持。

## 可维护性总结汇总
