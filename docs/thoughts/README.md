# 产品思考备忘

`docs/thoughts` 用来沉淀还没有成熟到 `docs/designs` 或 `docs/plans` 的产品、架构、交互和战略思考。

这里的文档应当记录有判断价值的讨论中间态：它已经超过 `docs/TODO.md` 的一句话想法，但还没有形成可执行计划或定稿设计。

## 适合放在这里

- 产品定位、长期方向、概念命名和方案空间讨论。
- 对某个用户场景的探索性拆解。
- 尚未定稿的架构取舍、产品边界和路线判断。
- 需要保留上下文，但暂时不适合进入 `docs/designs` / `docs/plans` 的讨论备忘。

## 不适合放在这里

- 一句话待办：放入 `docs/TODO.md`。
- 已经形成明确系统设计的方案：升级到 `docs/designs`。
- 已经准备执行的分步计划：升级到 `docs/plans`。
- 代码变更、发布、修复或验收记录：按 `nextclaw-iteration-log-governance` 判断是否进入 `docs/logs`。

## 命名

文件名必须使用 `YYYY-MM-DD-<kebab-topic>.thought.md`。
