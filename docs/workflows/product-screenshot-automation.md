# Product Screenshot Automation

## Goal

Automate product screenshots used by website and GitHub assets, replacing manual capture.

## Local Usage

1. Install dependencies:

```bash
pnpm install
pnpm exec playwright install chromium
```

2. Refresh screenshots with deterministic mock data:

```bash
pnpm screenshots:refresh
```

3. Refresh screenshots from a real local NextClaw instance when the result is used for website or product presentation:

```bash
SCREENSHOT_USE_REAL_APP_DATA=1 SCREENSHOT_UI_ORIGIN=http://127.0.0.1:5194 pnpm screenshots:refresh
```

Real app mode does not intercept `/api/*`, so the screenshots use the current local sessions, skills, providers, channels, panel apps, and other visible product data. Use this mode for public-facing website screenshots when a real local instance is available.

Website screenshots default to the `cool` theme (雾蓝) for a calmer, more consistent presentation. Override only when deliberately comparing themes:

```bash
SCREENSHOT_UI_THEME=natural pnpm screenshots:refresh
```

4. Use real marketplace data with mock app data (optional):

```bash
REAL_MARKETPLACE=1 pnpm screenshots:refresh
```

Optional override for remote source:

```bash
REAL_MARKETPLACE=1 REAL_MARKETPLACE_BASE=https://marketplace-api.nextclaw.io pnpm screenshots:refresh
```

When real marketplace fetch fails, script falls back to built-in mock data automatically.

- `images/screenshots/*`
- `apps/landing/public/nextclaw-chat-page-*.png`
- `apps/landing/public/nextclaw-agents-page-*.png`
- `apps/landing/public/nextclaw-providers-page-*.png`
- `apps/landing/public/nextclaw-channels-page-*.png`
- `apps/landing/public/nextclaw-cron-job-page-*.png`
- `apps/landing/public/nextclaw-skills-doc-browser-*.png`
- `apps/landing/public/nextclaw-skills-page-*.png`
- `apps/landing/public/nextclaw-panel-apps-page-*.png`
- `apps/landing/public/nextclaw-panel-app-running-*.png`
- `apps/landing/public/nextclaw-workspace-preview-*.png`

Some website screenshots are intentionally curated from a real local task rather than generated by the deterministic mock scene list. Keep their source copies in `images/screenshots/` and mirror them to `apps/landing/public/` when they are used by the landing page:

- `nextclaw-hero-workbench-*.png`
- `nextclaw-image-generation-result-*.png`

## CI Automation

Workflow: `.github/workflows/product-screenshots.yml`

- Trigger: `workflow_dispatch` and weekly schedule.
- Action: run `pnpm screenshots:refresh`.
- Result: auto-create PR if screenshot assets changed.

## Acceptance

After running refresh:

1. For website presentation updates, prefer `SCREENSHOT_USE_REAL_APP_DATA=1` and confirm every screenshot referenced by the page uses the same real data source, the default 雾蓝 theme, and non-empty representative content.
2. If the landing page references image-generation capability, confirm the asset shows the generated image result itself, not only an installation or configuration screen.
3. `git status` should only show screenshot asset updates (unless UI changed).
4. Key files should be regenerated:
   - `images/screenshots/nextclaw-chat-page-en.png`
   - `images/screenshots/nextclaw-agents-page-en.png`
   - `images/screenshots/nextclaw-providers-page-en.png`
   - `images/screenshots/nextclaw-channels-page-en.png`
   - `images/screenshots/nextclaw-cron-job-page-en.png`
   - `images/screenshots/nextclaw-skills-doc-browser-en.png`
   - `images/screenshots/nextclaw-skills-page.png`
5. Landing public mirrors should stay in sync with screenshot source set.
