# Platform Admin Classic Console Optimization Design

## 背景

`platform-admin` 虽然已经完成了基础壳层拆页，但当前结构仍然没有达到“经典后台控制台”的质量线。问题不在于有没有侧边栏，而在于我们把导航区写成了说明区，把页面结构写成了组件堆叠区，导致后台缺少顶级产品常见的稳定框架感。

用户反馈的核心不是“想要更花的视觉”，而是：

- 侧边栏应该只是导航，不应该塞大量描述文案
- 布局应该直接参考顶级后台产品的经典结构，而不是半自定义
- 视觉风格不要脱离 NextClaw 自身 UI，继续沿用产品现有暖中性色与克制面板风格
- 需要沉淀一批真正可复用的后台框架组件，而不是每个页面自己拼布局

## 长期目标对齐

这次优化直接服务 NextClaw 愿景中的三条主线：

- 统一入口：后台治理面必须像一个稳定控制面，而不是继续像临时运维页
- 统一体验：后台与主产品在配色、密度、表面样式上保持同一设计语言
- 自治控制面：平台管理能力要结构清晰、层次明确、可持续扩展

## 方案判断

### 方案 A：在现有壳层上继续做局部修补

优点：

- 改动更少

缺点：

- 无法根治“侧边栏过重、框架感不足、页面结构不统一”的问题
- 以后继续加页面时，还会再次漂移

### 方案 B：按经典后台模板重做框架层

即本次推荐方案：

- 视觉沿用 NextClaw UI 风格
- 信息架构直接收敛到经典后台骨架
- 壳层、页面框架、面板、工具栏统一沉淀为可复用组件

优点：

- 能一次性解决结构问题
- 后续扩页时有稳定模板
- 用户感知会直接从“像拼出来的页”升级为“正式控制台”

缺点：

- 需要对壳层和几个主页面做一次结构重排

## 顶级产品共通模板

这次不抄具体视觉稿，只抄顶级产品共同骨架：

1. 左侧固定导航
2. 顶部固定全局栏
3. 内容区独立滚动
4. 侧边栏只承担导航职责
5. 页面统一遵守：
   - 顶部标题区
   - 工具栏区
   - 内容区
6. 内容区只允许稳定模块：
   - 指标卡
   - 表格
   - 筛选条
   - 双栏详情
   - 表单面板

## 视觉策略

本次不重新定义后台视觉风格，直接继承 NextClaw UI 的现有语言：

- 暖中性色背景
- 白色主内容面板
- 克制的阴影与边框
- Olive Green 品牌主色
- 更紧凑的间距和更克制的圆角

明确不做：

- 大面积渐变 Hero
- 夸张大圆角介绍卡
- 营销页式文案块
- 在侧边栏堆解释性内容

## 组件化重构目标

本次要补齐一组后台框架组件：

- `AdminShell`
  - 固定侧边栏
  - 固定顶部栏
  - 内容区滚动
- `AdminPage`
  - 页面标准容器
- `AdminToolbar`
  - 搜索、筛选、分页控制区
- `AdminSurface`
  - 统一卡片/面板表面
- `AdminMetricCard`
  - 统一指标卡

规则：

- 侧边栏只展示菜单项，不展示长描述
- 当前页面说明只在顶部栏保留一行
- 页面内容必须在统一容器内排布，不能再自由生长

## 页面结构重做

### 1. 壳层

- 左侧导航宽度固定
- 顶部栏高度固定
- 内容区内部滚动
- 管理员信息收敛到侧边栏底部的紧凑信息区

### 2. 总览页

经典驾驶舱结构：

- 第一行：关键指标卡
- 第二行：Remote 配额面板
- 第三行：营收与上游治理

删除大 Hero 入口卡和夸张说明块。

### 3. Marketplace 审核页

保留当前正确的审核工作台结构，但收回到标准页面框架：

- 顶部栏负责页面标题
- 页面主体保留：
  - 状态摘要
  - 筛选条
  - 左侧队列
  - 右侧详情

### 4. 用户与额度页

- 上方：全局额度治理
- 下方：用户额度表格

以“管理台”而不是“说明卡片集合”的方式呈现。

### 5. 充值审核页

- 顶部：状态切换
- 主体：审核表格
- 底部：分页控制

## 验证与发布

本次收尾要求：

1. 本地静态检查
   - `pnpm -C apps/platform-admin tsc`
   - `pnpm -C apps/platform-admin lint`
   - `pnpm -C apps/platform-admin build`
2. 本地 UI 冒烟
   - `PLATFORM_ADMIN_BASE_URL=http://127.0.0.1:4177 pnpm smoke:platform:admin`
3. 发布
   - `pnpm deploy:platform:admin`
4. 线上 UI 冒烟
   - `PLATFORM_ADMIN_BASE_URL=https://platform-admin.nextclaw.io pnpm smoke:platform:admin`

## 非目标

本次不做：

- 新增二级导航
- 重写后台 API
- 新增 Remote / 设置独立页
- 重做数据模型
- 大规模视觉改版

本次只解决一件事：

把 `platform-admin` 的框架层做成真正经典、稳定、可复用的后台控制台模板。
