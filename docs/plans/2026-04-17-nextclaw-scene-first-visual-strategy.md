# NextClaw 场景优先宣传图策略

## 这次纠偏的核心

上一轮宣传图思路的问题，不只是“图不像 NextClaw”，还有一个更深的方向错误：

**我们太先讲产品了，没有先讲用户场景。**

对大多数用户，尤其是非技术用户来说，他们不会先问：

- 你的架构是什么
- 你是不是统一入口
- 你是不是个人 AI 操作层

他们先问的是：

- 这玩意能帮我干啥
- 它能不能替我省事
- 它能不能进入我现在的工作和生活
- 它是不是比我现在来回切工具更方便

所以首批传播图的第一原则应该改成：

**先让用户一眼看到“它能替我做什么”，再让用户理解“它为什么这么强”。**

## 传播顺序重排

错误顺序：

1. 讲抽象产品定位
2. 讲统一入口
3. 讲架构价值
4. 用户自己脑补应用场景

正确顺序：

1. 先给用户一个具体场景
2. 让用户马上明白自己能得到什么
3. 再补一句为什么 NextClaw 能做到
4. 最后再升维到“统一入口 / 个人 AI 操作层”

## 首图标准

一张好的 NextClaw 宣传图，必须让普通用户在 1 到 2 秒内看懂以下任一件事：

- 我可以在微信/飞书/聊天入口里直接让 AI 帮我做事
- 我不用切一堆模型和工具，它帮我统一起来了
- 它可以替我定时盯任务、跑流程、持续帮我干活
- 它不是只能聊天，而是能进入真实工作流
- 它本地可控，但我不必被一台电脑困住

如果一张图只让人感到“科技感很强”，但看不出“具体能干啥”，那它不是合格首图。

## 用户真正会被打动的 5 个场景

### 1. 沟通入口场景

一句话：

`让 AI 直接进入你每天真实在用的沟通入口。`

用户秒懂点：

- 不用专门打开一个新系统
- 在微信、飞书、Discord 里就能让 AI 干活
- AI 开始像一个真实可用的助手，而不是网页里的聊天玩具

适合画面：

- 左边是用户熟悉的沟通入口
- 中间是 NextClaw 中枢
- 右边是 AI 实际做出来的结果或动作

### 2. 多工具统一场景

一句话：

`不用自己切模型、切渠道、切配置，一个地方统一搞定。`

用户秒懂点：

- 省掉来回切换
- 不用理解复杂技术概念
- 有“省脑子”的直接价值

适合画面：

- 一个清晰的 NextClaw 控制台
- 周围多个工具/模型/渠道被收拢进来
- 视觉重点不是“多”，而是“被统一”

### 3. 自动化代劳场景

一句话：

`不是你每次问它，它才动；而是它能按计划持续替你做事。`

用户秒懂点：

- 提醒、巡检、跟进、定时处理
- AI 不再只是聊天，而是开始帮你执行
- 直接指向“省时间”

适合画面：

- 时间线 / 定时任务 / 周期执行
- NextClaw 在后台持续跑
- 用户只看到结果被交付出来

### 4. 跨设备可达场景

一句话：

`本地运行，但不被本机困住。`

用户秒懂点：

- 安全可控
- 但又不是只能坐在这台电脑前
- 能从别的设备继续用

适合画面：

- 本地主机作为可信中枢
- 浏览器 / 手机 / 远程入口作为延展
- 强调“可控”和“可达”

### 5. 工作流中枢场景

一句话：

`它不是陪你聊，而是开始进入你的真实工作流。`

用户秒懂点：

- 收消息、调模型、跑任务、交付结果
- 它像一个会干活的中枢，而不是回答机器

适合画面：

- 一个任务从输入到执行到结果的短链路
- NextClaw 位于中间，负责连接和调度

## 首批应该先做哪类图

如果只能先做 3 张，我建议优先顺序是：

1. `沟通入口场景图`
2. `自动化代劳场景图`
3. `多工具统一场景图`

原因：

- 这三张最容易让非技术用户秒懂
- 这三张最容易让用户产生“哦，这对我有用”的感觉
- 这三张天然比“架构图”更适合传播

## 每张图的正确问法

以后做任何宣传图，不要先问：

- 这张图怎么更酷
- 这张图怎么更像平台
- 这张图怎么体现统一入口

而要先问：

- 用户看到这张图，能不能立刻说出它能干啥
- 用户会不会立刻联想到自己的某个场景
- 用户看完会不会产生“这能替我省事”的感觉
- 就算不看正文，这张图能不能单独成立

## Prompt 写法也要改

以后不要再以这种方式开头：

- create a premium hero image for an AI operating layer

而要改成这种方式：

- show a real everyday user scenario where NextClaw helps someone get work done inside familiar communication tools
- show how NextClaw saves the user from switching between models, tools, and channels
- show NextClaw running scheduled tasks and delivering results automatically

也就是说，prompt 的第一句不该先是“产品是什么”，而该先是“用户正在经历什么，NextClaw 替他解决了什么”。

## 一张图的理想信息结构

最推荐的结构是：

1. 用户熟悉的场景入口
2. NextClaw 在中间承担动作和调度
3. 结果被清晰交付
4. 再补一句短标题，命名这件事

例如：

- 左边：微信/飞书消息
- 中间：NextClaw 控制和执行
- 右边：任务结果、自动回复、整理好的内容

这样用户看到图时，不需要先理解技术概念，就能理解价值流。

## 直接可用的标题方向

适合小白用户的一句话，不该太技术化。

更好的方向：

- `让 AI 进入你每天真实在用的沟通入口`
- `不用切来切去，一个地方把 AI 跑起来`
- `不是只会聊天，而是能替你持续做事`
- `本地可控，也能随时继续用`
- `把模型、消息和任务收成一个会干活的入口`

不推荐首屏直接使用：

- `Personal AI Operating Layer`
- `Unified Orchestration Console`
- `Agentic Runtime Control Plane`

这些词可以作为第二层解释，不能作为第一眼卖点。

## 对首图的新验收标准

首图通过的标准，不再是“有没有科技感”，而是下面四条：

1. 非技术用户能一眼看懂一个具体场景
2. 用户能说出一个明确收益
3. 用户会觉得“这个和我有关”
4. 图和产品本身有真实关系，不是泛 AI 海报

## 下一步执行建议

接下来宣传图应该分成两层：

### 第一层：场景首图

负责：

- 抓住眼球
- 讲清用途
- 面向普通用户

### 第二层：结构解释图

负责：

- 讲清为什么它能做到
- 承接进一步理解
- 面向更愿意深挖的人

也就是说：

- 第一层回答：`它能帮我干啥`
- 第二层回答：`它为什么这么强`

## 长期目标对齐 / 可维护性推进

这次纠偏是必要的。

推进点在于：

- 把传播重心从“产品自我表达”拉回“用户场景理解”
- 防止继续产出看起来很高级、但无法转化的泛 AI 海报
- 让后续所有视觉资产都先经过“用户能否秒懂用途”的筛选

本次不涉及代码改动，因此代码可维护性评估不适用。
