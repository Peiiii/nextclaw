# NextClaw 首批两张宣传图执行 Brief

## 目的

这份文档承接 [2026-04-17-nextclaw-marketing-visual-asset-plan.md](./2026-04-17-nextclaw-marketing-visual-asset-plan.md)，把首批最关键的两张图直接推进到可执行层。

目标不是继续抽象讨论“该做什么图”，而是直接明确：

- 每张图到底要讲什么
- 画面该怎么构
- 文案该怎么落
- 哪些元素必须出现
- 哪些元素必须避免
- 如果用 AI 生成图，应该怎么写 prompt

## 总体方向

这两张图不是为了追求单纯炫酷，而是承担整个产品传播系统的底座。

排序优先级：

1. 先讲清楚
2. 再讲高级感
3. 最后才是酷

统一视觉要求：

- 未来感，但不是廉价赛博风
- 控制台感，但不是纯工程截图拼贴
- 有产品质感，但不空喊概念
- 必须保留真实 UI 或真实能力结构的影子

统一版式原则：

- 一张图只讲一个核心结论
- 标题必须能单独成立
- 副标题只补价值，不重复标题
- 画面中间必须存在“NextClaw 作为中枢”的视觉锚点
- 不允许把一堆 logo 平铺当信息结构

## 图一：总价值海报

### 一句话任务

让用户第一次看到这张图时，立刻明白：

**NextClaw 不是一个聊天壳，而是个人 AI 操作层入口。**

### 这张图最该解决的问题

当前很多用户即使看过 README，也更容易记住“多模型”“多渠道”“本地运行”这些分散能力，而不是记住它们为什么会被放在同一个产品里。

这张图的任务就是把这些点收成一个认知：

- 统一入口
- 统一控制
- 统一调度
- 统一工作流

### 适用位置

- 官网首屏主视觉
- README 主图
- V2EX / 长帖首图
- GitHub 社交卡片基础版本

### 目标受众

- 第一次认识 NextClaw 的用户
- 看过类似产品但尚未建立差异认知的人
- 对“本地 AI / agent / automation / multi-channel”有兴趣的人

### 标题候选

优先候选：

- `NextClaw 是你的个人 AI 操作层`

备选：

- `一个入口，接管你的 AI 世界`
- `不是聊天壳，是你的 AI 操作层入口`
- `One Command. One Console. Your AI World.`

### 副标题候选

优先候选：

- `把模型、渠道、自动化、远程访问和工作上下文收进一个统一入口。`

备选：

- `Run locally. Reach anywhere. Orchestrate everything.`
- `从聊天到自动化，从本地到远程，一个控制台统一完成。`

### 画面结构

建议构图：

- 画面中心是一个带强存在感的 NextClaw 主控制台或抽象化 UI 面板
- 控制台不是完整截图平铺，而是经过设计处理的产品面板组合
- 四周以半环绕或放射状连接几个高价值能力节点：
  - Providers
  - Channels
  - Automation
  - Remote Access
  - Project Context
- 节点之间要体现“统一调度”关系，而不是五张卡片并列陈列

### 视觉氛围

- 深色背景
- 低噪点渐变或空间层次背景
- 面板边缘带轻微辉光
- 连接线和节点状态体现“控制中枢”的秩序感
- 不要满屏炫光，不要太像游戏海报

### 必须出现的产品信息

- NextClaw 品牌名
- 统一入口这个核心认知
- 至少 4 个能力锚点：
  - 多模型
  - 多渠道
  - 自动化
  - 远程访问

### 可以出现但不强制的元素

- `nextclaw start`
- 浏览器窗口轮廓
- 多设备轮廓
- 会话 / project context 的轻量提示

### 必须避免

- 把所有渠道 logo 全塞上去
- 画成“宇宙 + 神经网络 + 蓝紫光”
- 标题太抽象，看不出产品价值
- 完全没有产品面板，像纯品牌海报

### 首屏信息层级

从上到下：

1. 品牌
2. 标题
3. 副标题
4. 主视觉中枢
5. 四到五个能力锚点

### 推荐尺寸

- `1792 x 1024`：官网 / README 横版主图
- `1600 x 900`：社交分享横图
- `1080 x 1350`：社媒竖版裁切版

## 图二：统一入口解释图

### 一句话任务

把 NextClaw 的价值讲明白，不靠口号，靠结构。

### 这张图最该解决的问题

很多人会把 NextClaw 理解成：

- 一个能接多个模型的 UI
- 一个能接很多渠道的聊天系统
- 一个本地运行的 agent 工具

这些都不算错，但都不够。

这张图要把“为什么这些能力必须在同一个入口里”可视化表达出来。

### 适用位置

- 官网中段解释图
- 文档首页
- 长帖正文配图
- 对外介绍 deck

### 目标受众

- 已经对产品产生初步兴趣的人
- 想进一步理解产品结构和价值的人
- 需要向别人转述 NextClaw 是什么的人

### 标题候选

优先候选：

- `一个入口，连接模型、渠道、自动化与工作流`

备选：

- `NextClaw 如何把 AI 能力收成一个统一控制台`
- `不是拼插件，而是统一调度`

### 副标题候选

优先候选：

- `用户先进入 NextClaw，再由 NextClaw 去连接模型、消息入口、自动化和远程工作流。`

### 画面结构

建议采用信息图 + 产品面板融合形式，而不是纯摄影海报。

核心结构：

- 正中心：NextClaw 中枢面板
- 左上：Providers
- 左下：Channels
- 右上：Automation
- 右中：Remote Access
- 右下：Project Context / Sessions
- 中心到外围用有秩序的连线、箭头或状态轨迹表达

建议每个模块都不是只写一个名词，而是带一个简短价值标签：

- Providers：`Switch and orchestrate models`
- Channels：`Bring AI into real communication flows`
- Automation：`Run tasks on schedule`
- Remote Access：`Reach your AI beyond one device`
- Project Context：`Keep work grounded in real sessions`

### 风格建议

- 比图一更理性、更结构化
- 减少情绪化视觉，增强图解表达
- 适合做成长图、官网解释区、文档导图

### 必须出现的产品信息

- “用户先进入 NextClaw”的中枢感
- 外围五类能力模块
- 每类模块对应的用户价值短句

### 必须避免

- 纯架构图气质过重
- 只有框线没有视觉吸引力
- 文字太多，像 PPT 页面截图
- 全部使用专业术语，普通用户看不懂

### 推荐尺寸

- `1600 x 1200`：官网 / 文档解释图
- `1080 x 1350`：长图帖版本
- `1920 x 1080`：演讲 / 视频封面版本

## 两张图的关系分工

图一负责：

- 打第一印象
- 建立心智
- 激发点击和停留

图二负责：

- 把价值讲透
- 承接进一步理解
- 帮助用户形成复述能力

如果只看图一，用户会记得“这是个高级的统一入口产品”。

如果连图二一起看，用户会进一步明白：

**为什么它不是功能堆砌，而是一个真正的产品化操作层。**

## 推荐落地顺序

1. 先出图一的 2 到 3 个构图方向
2. 确认品牌气质后再出图二
3. 图一确定后同步替换 README 主视觉方向
4. 图二确定后同步给官网中段和长帖使用
