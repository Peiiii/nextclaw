# NextClaw 宣传视觉资产与传播系统首版方案

## 背景

当前 NextClaw 已经有一定的产品截图、README 首屏图、社区帖子与对比材料，但整体仍停留在“有素材”而不是“有传播系统”的阶段。

现在最突出的问题不是文案数量不够，而是缺少一套能够稳定承担以下任务的视觉传播资产：

- 一眼讲清产品到底是什么
- 一眼讲清为什么值得试
- 一眼讲清和普通聊天壳、单点 agent 工具的区别
- 能在社交平台上被停留、被转发、被复述

换句话说，NextClaw 现在更像“产品在持续变强”，但还不像“传播机器已经成形”。

## 深层意图判断

这次要解决的并不是“补几张宣传图”的表层问题，而是补齐一整套产品对外表达基础设施：

- 让用户第一次看到 NextClaw 时能快速建立正确心智
- 让不同渠道的内容开始共享统一视觉风格与统一叙事
- 让传播不再依赖临时发挥，而是进入可批量生产、可持续迭代的状态

## 核心判断

NextClaw 最应该被传播的，不是“功能很多”，而是：

**它正在成为 AI 时代的个人操作层入口。**

这句大话不能直接硬喊，必须被拆成用户一眼能看懂的几个可视化价值点：

- 一个入口，统一模型、渠道、自动化、远程访问和工作上下文
- 本地优先，但不被单机绑死
- 不是聊天壳，而是能进入真实工作流的 AI 中枢
- 比厚重平台更轻、更快开始用、更容易长期维护

因此，传播资产的目标不是“做得酷”，而是先做到：

1. 看得懂
2. 记得住
3. 愿意转
4. 能复用

## 总目标

在未来一轮传播周期内，先建立一套可持续复用的基础传播资产系统，而不是零散地做单张图。

第一阶段目标：

- 建立统一视觉母版与信息排版规则
- 做出首批高复用传播图
- 让 README、官网、V2EX、X、社群封面开始共享同一套视觉语言
- 让产品价值表达从“功能清单”升级为“心智 + 场景 + 证据”

## 传播策略总纲

### 一、主叙事

所有视觉资产默认围绕同一条主线展开：

**NextClaw 不是又一个 AI 聊天壳，而是把模型、渠道、自动化、远程访问和项目上下文收进一个统一入口的个人 AI 操作层。**

### 二、三层传播结构

#### 1. 认知层

回答“这是什么”。

适合用：

- 价值解释图
- 一页产品定位图
- 对比认知图

#### 2. 代入层

回答“这东西怎么进入我的生活或工作”。

适合用：

- 场景海报
- 工作流图
- 多入口使用场景图

#### 3. 扩散层

回答“我为什么愿意停下来、截图、转发、评论”。

适合用：

- 炫酷封面图
- 短动效海报
- 数据 / 对比 / 反差类图卡

## 视觉资产体系

### A. 基础品牌视觉母版

作用：

- 统一所有传播图的色彩、字体、网格、标题样式、按钮样式、边框语言
- 避免每次都像临时拼贴

建议方向：

- 基调：产品化、未来感、控制台感、非廉价赛博风
- 气质：清晰、强控制感、轻克制、可信赖
- 避免：过重的 AI 幻觉感、抽象宇宙背景、纯概念海报、只有光效没有信息

建议元素：

- 深色中性底 + 明亮高对比强调色
- 终端 / 面板 / 卡片 / 节点连线语言
- 具有“统一调度中心”意味的构图
- 适度发光，不做廉价霓虹堆砌

### B. 价值解释型资产

作用：

- 一眼讲清楚 NextClaw 的产品价值
- 优先服务官网首屏、README 首图、介绍帖首图

适合内容：

- 一个入口接多个世界
- 本地优先 + 跨设备可达
- AI 不只是聊天，而是工作流入口

### C. 场景代入型资产

作用：

- 用真实场景激发代入感
- 让用户不是“理解功能”，而是“看见自己会怎么用”

适合内容：

- 微信 / 飞书 / Discord 同时接入
- 浏览器里统一配置模型和渠道
- 本地运行但可远程打开
- 自动化定时执行

### D. 对比认知型资产

作用：

- 快速帮用户建立差异认知
- 降低“看起来都差不多”的认知阻力

适合内容：

- NextClaw vs 聊天壳
- NextClaw vs 厚重平台
- NextClaw vs 单点 coding agent 工具

### E. 扩散型资产

作用：

- 用好看、锋利、有记忆点的图承担平台扩散
- 更适合 X、社群转发、帖子封面

适合内容：

- “一个入口，接管你的数字世界”
- “把 AI 放进微信、飞书、终端和浏览器”
- “不是多一个工具，而是多一个操作层”

## 第一批必须先做的 6 张图

### 1. 总价值海报

- 暂定标题：`NextClaw 是你的个人 AI 操作层`
- 目的：承担整个产品的总心智图
- 核心信息：
  - 一个入口
  - 多模型
  - 多渠道
  - 自动化
  - 远程访问
  - 本地优先
- 适用渠道：
  - 官网首屏
  - README 主图
  - V2EX 帖子首图

### 2. 统一入口解释图

- 暂定标题：`一个入口，连接模型、渠道、自动化与工作流`
- 目的：解释 NextClaw 不是单点功能
- 画面建议：
  - 中间是 NextClaw 控制台
  - 外围连接 Provider、微信/飞书/Telegram、Cron、Remote Access、Project Context
- 适用渠道：
  - 官网中段
  - 文档首页
  - 长帖配图

### 3. 场景海报：AI 进入真实沟通入口

- 暂定标题：`让 AI 进入你真实在用的沟通入口`
- 目的：放大多渠道的直观价值
- 画面建议：
  - 微信、飞书、Discord、Email 等入口围绕统一中枢
  - 避免简单 logo 堆砌，要体现“同一 agent 跨入口工作”
- 适用渠道：
  - 中文社区
  - 微信群 / Discord 宣传
  - 国内渠道帖子

### 4. 场景海报：本地优先但不困在本机

- 暂定标题：`本地运行，但不被单机困住`
- 目的：放大远程访问与跨设备能力
- 画面建议：
  - 一台本地机器作为主节点
  - 浏览器、手机、远程入口作为延展
  - 强调“可控”和“可达”同时成立
- 适用渠道：
  - 官网
  - V2EX
  - 英文社媒

### 5. 对比图：不是聊天壳

- 暂定标题：`NextClaw 不只是聊天壳`
- 目的：建立差异认知
- 对比维度建议：
  - 单点聊天
  - 单点 coding agent
  - 厚重平台
  - NextClaw
- 适用渠道：
  - 对比页
  - FAQ
  - 长图帖

### 6. 炫酷扩散海报

- 暂定标题：`One Command. One Console. Your AI World.`
- 目的：承担社交平台停留与转发
- 画面建议：
  - 更强调构图、质感、氛围
  - 文案极短
  - 保留明确产品截图或面板轮廓，避免完全抽象
- 适用渠道：
  - X
  - GitHub 社交卡片
  - 社群封面

## 视觉创意原则

### 必须坚持

- 信息密度高，但阅读路径清晰
- 图里必须有“产品价值信息”，不是只做情绪壁纸
- 每张图只能有一个主结论
- 图和文案必须让人能一句话复述
- 截图要服务价值表达，不是机械拼屏

### 明确避免

- 把多个卖点塞进一张图导致失焦
- 纯 logo 拼贴
- 纯赛博光效海报，没有产品信息
- 用空洞口号替代真实价值
- 所有图都长得像同一张版式的换字版本

## 2 周执行节奏

### 第 1 周

- 定视觉母版
- 定标题体系
- 定 6 张图的核心文案
- 先完成 2 张最关键图：
  - 总价值海报
  - 统一入口解释图

### 第 2 周

- 完成剩余 4 张图
- 同步替换 README / 官网 / 社区封面
- 从 6 张图里拆出：
  - 3 张长帖配图
  - 4 张社媒图卡
  - 1 张 GitHub 社交分享图

## 执行角色建议

如果按“营销负责人 + 设计执行 + 内容分发”三段来组织，建议职责如下：

- 营销负责人：
  - 统一主叙事
  - 审核是否讲清价值
  - 决定优先级
- 视觉设计：
  - 统一母版
  - 完成海报与图卡
  - 保证审美与识别度
- 内容分发：
  - 把一张图拆成不同平台版本
  - 配套标题、文案、CTA
  - 记录各渠道效果

如果资源有限，优先级应该是：

1. 先做主叙事和图的结构
2. 再做精细视觉
3. 最后再扩充数量

## 成功标准

如果这套方案有效，应该出现这些变化：

- 用户第一次看到图时，能更快说出“这产品到底是干什么的”
- README / 官网首页不再只是截图陈列
- 社区帖子更容易被点开和转述
- X 或社群封面不再像临时活动海报
- 一套图能被多次复用，而不是一次性消耗品

## 现在不建议优先做的事

- 先花很长时间做大而全品牌手册
- 先做大量零散平台封面图
- 先重做 logo
- 先做纯炫技 CG 海报
- 先铺十几个渠道但每个都没有稳定资产

当前最该做的是：

**先把“可解释、可复用、可扩散”的首批核心传播图做出来。**

## 长期目标对齐 / 可维护性推进

这次方案顺着长期方向推进了一小步。

推进点在于：

- 让 NextClaw 的传播开始更贴近“个人操作层”而不是“功能清单”
- 让后续营销资产有统一骨架，减少临时拼图式输出
- 先用少量高复用核心资产建立系统，再逐步扩展，避免传播资产无边界膨胀

本次不涉及代码可维护性评估，因此该项对代码层面为“不适用”。

但从传播系统维护角度，这版方案优先遵循了“删减优先、简化优先”的原则：

- 不主张一开始做大量海报
- 不主张同时铺满所有平台
- 不主张先做复杂品牌工程
- 先收敛出 6 张高复用核心图，作为后续所有版本的母体

下一步最合适的推进方式，不是继续抽象讨论，而是直接把首批 6 张图中的前 2 张进入设计稿阶段。
